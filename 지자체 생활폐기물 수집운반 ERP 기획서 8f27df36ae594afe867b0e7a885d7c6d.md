# 지자체 생활폐기물 수집운반 ERP 기획서

👋 전체 시스템 구조, DB 설계, 기능 정의를 체계적으로 정리

# 1. 시스템 전체 구조 개요

[슈퍼관리자] → 지자체 생성
↓
[지자체 관리자] → 위탁업체 등록
↓
[위탁업체 관리자] → 청소구역/행정동 설정 → 내부관리자/근로자 계정 생성
↓
[내부관리자] → 업무 승인/정산/보고
[근로자] → 웹앱으로 근태/민원/안전/차량 입력

## 2. DB 구조 설계

### 2-1. 테넌트(Tenant) 계층 구조

지자체(Municipality) → 위탁업체(Contractor) → 내부조직(Department) → 근로자(Worker)
**멀티테넌트 아키텍처**로 설계하는 것이 핵심. 총괄관리자 지자체명 입력 → 위탁업체 서비스 생성

### 2-2. 핵심 테이블 설계

- - 지자체 테이블
CREATE TABLE municipalities (
id BIGINT PK AUTO_INCREMENT,
name VARCHAR(100) NOT NULL, -- 예: 서울특별시 강남구
code VARCHAR(20) UNIQUE NOT NULL, -- 행정코드: 기존 db에서 받아 옴
status ENUM('PENDING','ACTIVE','SUSPENDED'),
created_at DATETIME,
created_by BIGINT -- 슈퍼관리자 ID → 서버관리자
);
- - 위탁업체 테이블
CREATE TABLE contractors (
id BIGINT PK AUTO_INCREMENT,
municipality_id BIGINT FK → [municipalities.id](http://municipalities.id/),
company_name VARCHAR(100),
business_no VARCHAR(20) UNIQUE, -- 사업자번호
contract_start DATE,
contract_end DATE,
status ENUM('SETUP','ACTIVE','EXPIRED'),
-- SETUP: 지역/행정동 입력 중
-- ACTIVE: 정상 운영
db_schema_name VARCHAR(50), -- 업체별 스키마 분리 시
created_at DATETIME

CREATE TABLE cleaning_zones (

CREATE TABLE admin_dongs (

### 2-3. 근태관리 테이블 (중요)

CREATE TABLE attendance_records (

### 3-2. 근태관리 — 관리자 추가 조정/수정 기능 (핵심)

근태관리에서 관리자가 **반드시** 처리해야 할 추가 기능들

A. 근태 조정 필수 기능

기능	설명	비고
출퇴근 시간 수정	기기 오류, 망각 등으로 인한 시간 수정	사유 입력 필수
근무 유형 변경	정상 → 연장/야간/휴일 변경	임금 계산에 직결
결근 처리	무단결근 vs 유급결근 구분 처리	인건비 정산 연계 
일괄 등록	외부행사/교육 등 단체 출근 일괄처리	다수 근로자 동시
대리 출근 등록	근로자 대신 관리자가 출근 등록	특수상황 처리
월 마감 확정	월별 근태 최종 확정 (이후 수정 잠금)	정산 전 필수
소급 수정	이전 월 근태 수정 (권한 상위자만 가능)	이중승인 필요

B. 임금 계산 연계 조정

기본급 산정: 생활페기물 수집운반 원가산정 규정 적용(기후에너지환경부)

- 소정근로시간 기준 (월 209시간 기준)
- 결근공제 = 기본급 ÷ 소정근로일수 × 결근일수

가산임금 자동 계산: 근로기준법, 노동법

- 연장근로 (8시간 초과): × 1.5
- 야간근로 (22:00~06:00): × 0.5 추가
- 휴일근로: × 1.5 (8시간 초과 시 × 2.0)

관리자 조정 항목:

- 식대/교통비 지급 여부
- 특수수당 (위험수당, 기술수당)
- 연차수당 산입 여부

### C. 근태 이상 알림 기능 (관리자 대시보드)

- **미출근 알림**: 출근 시간 30분 경과 시 알림
- **미퇴근 알림**: 퇴근 예정 시간 초과 시 알림
- **GPS 이탈 경고**: 담당 구역 외 출근 시 플래그
- **연속 야간/연장 경고**: 노동법 위반 가능성 사전 경고
- **월 연장근로 52시간 초과 경고**

D. 근태 감사추적 (Audit Trail) → 향후, 스마트워치 GPS연동 가능 있음.

모든 수정 이력은 삭제 불가능하게 보존해야 합니다. 노동청 감사 대비.

- - 모든 수정은 attendance_adjustments 테이블에 이력 보존
-- 수정자, 수정 시각, 수정 사유, 원본값, 변경값 전부 기록
-- 월 마감 후 수정 시 상위 관리자 이중 승인 필요

### 3-3. 민원업무 (웹앱)

근로자 입력 항목:

- 민원 접수 일시

- 민원 배정 (담당자 지정)
- 처리 기한 설정
- 처리 현황 모니터링
- 지자체 보고서 자동 생성(민원일지) → form 적

### 3-4. 산업안전보건업무 (웹앱)

근로자 입력 항목:

- 안전점검 일일 체크리스트 (차량, 장비, 보호구)
- 아차사고 보고 (위험상황 발견)
- 재해 발생 보고 (사고 즉시)
- 안전교육 이수 확인 서명(TBM)

관리자 관리:

- 안전점검 현황 모니터링(출퇴근 공지관리, 일자별 기상 및 온도 조회, 기상안전 관리 → 이력 관리)
- 위험 요소 집계 및 분석(건강기록카드 작성 및 관리 → 업체 관리자 만 조회, 119 및 비상연락처 → 위급시 자동 앱에서 긴급상황 문자 전송)
- 재해 통계 (연도별/월별)
- 안전교육 계획 및 이수율 관리 - 작업안전가이드 열람 및 근로자 서명
- 고용노동부 보고 자료 자동 생성

## 4. 계정 생성 흐름 (온보딩 프로세스)

Step 1. 슈퍼관리자 → 지자체 생성 (지자체명, 코드, 담당자)
Step 2. 슈퍼관리자 → 지자체 관리자 계정 생성
Step 3. 지자체 관리자 → 위탁업체 등록 (사업자번호, 계약기간)
Step 4. 위탁업체 관리자 계정 자동 생성 & 이메일 발송
Step 5. 위탁업체 관리자 로그인 → 청소구역/행정동 입력 [SETUP 상태]
Step 6. 청소구역 입력 완료 → 서 관리자 승인
Step 7. 승인 완료 → DB 활성화 [ACTIVE 상태]
Step 8. 위탁업체 관리자 → 내부관리자/근로자 계정 발급 시작
Step 9. 근로자 → 웹앱 설치 후 업무 시작

## 5. 추가 권고 사항

**기술 스택 추천: 영역(백엔드, 프론트엔드(관리자), 추천 기술, 이유)**

**멀티테넌트 DB 전략 추천 :** 
업체 수가 많아질 경우를 고려하여 **단일 DB + contractor_id 컬럼 기반 Row-level 격리** 방식 고려

**법적 필수 준수 사항**

- **근로기준법**: 연장/야간/휴일 가산임금 정확 계산
- **개인정보보호법**: 근로자 GPS 정보 수집 동의, 암호화 저장
- **산업안전보건법**: 재해 발생 시 고용노동부 보고 기한 (사망 1일, 부상 1개월)
- **공공기관 보안**: CC 인증 여부, 보안 취약점 점검 필요

## 7. 지자체 관리자 전용 화면 및 권한 설계

#### 7-1. 권한 매트릭스 업데이트 (지자체 중심)

지자체는 관리/감독의 주체이므로, 데이터의 생성 및 수정 권한은 위탁업체에 두고 지자체는 결과 데이터를 투명하게 들여다보고 정산에 필요한 서류만 확보하는 구조로 권한(Role-Based Access Control, RBAC)을 재설계

#### 7-2. 지자체 전용 모니터링 및 다운로드 기능 설계

지자체 관리자가 로그인했을 때 보게 될 **'지자체 전용 대시보드'** 와 각 메뉴별 세부 기능입니다.

### A. 민원관리 모니터링 (Read-Only & Export)

- **대시보드 뷰**: 행정동별/위탁업체별 민원 접수 및 처리 현황 그래프 (원형/막대 그래프).
- **리스트 뷰**: 전체 민원 목록 조회 (민원유형, 발생위치, 처리상태, 처리전후 사진 확인 가능).
- **제한 사항**: '담당자 배정', '처리 상태 변경', '내용 수정/삭제' 버튼 완전 비노출.
- **파일 다운로드**:
    - **Excel 다운로드**: 기간별 민원 처리 대장 (행정감사용 데이터).
    - **PDF 다운로드**: 개별 민원 처리 결과 보고서 (사진 포함).

### B. 차량운행일지 모니터링 (Read-Only & Export)

- **대시보드 뷰**: 업체별 차량 가동률, 총 주행거리, 총 수거량(Ton) 추이.
- **리스트 뷰**: 차량별 일일/월간 운행일지 리스트 조회.
- **제한 사항**: 운행일지 '신규 작성', '수정', '결재' 기능 접근 불가.
- **파일 다운로드**:
    - **Excel 다운로드**: 월간 차량별 운행 기록부 (주행거리, 유류사용량, 수거량 포함).
    - *(선택)* 지자체 제출용 운행일지 표준 양식에 맞춘 엑셀 다운로드 지원.

### C. 노무비 정산 모니터링 (Read-Only & Export)

- **대시보드 뷰**: 당월 총 노무비 청구액, 위탁업체별 예산 소진율, 전월 대비 증감액.
- **리스트 뷰**: 위탁업체가 '최종 제출(확정)'한 노무비 청구 내역서 및 개인별 지급 명세(이름 등 민감정보는 마스킹 처리 여부 결정 필요).
- **제한 사항**: 근로자의 원본 출퇴근 시간 수정 불가, 가산수당 비율 조정 불가, 결재/반려 기능 외 데이터 조작 불가.
- **파일 다운로드**:
    - **Excel/PDF 다운로드**: 노무비 청구서, 임금대장(지자체 제출용), 개인별 이체 내역 확인서.
    - 원가산정 보고서 전체 다운로드.

#### 7-3. 시스템 구현 시 기술적 주의사항(보안 및 UI)

1. **UI/UX 레벨 (프론트엔드)**
- 로그인한 계정의 Role이 `MUNI_ADMIN`(지자체 관리자)일 경우, 화면 상의 모든 `<button>` (저장, 수정, 삭제, 신규등록)을 숨김 처리(Hidden) 하거나 비활성화(Disabled)
- 오직 `[목록]`, `[상세보기]`, `[엑셀 다운로드]`, `[PDF 다운로드]` 버튼만 활성화
1. API 레벨 (백엔드)
- 지자체 관리자의 API 호출은 **데이터 조회용인 `GET` 요청만 허용**
- 혹시라도 비정상적인 방법으로 데이터를 추가/수정하는 `POST`, `PUT`, `DELETE` 요청이 들어오면 서버에서 `HTTP 403 Forbidden(권한 없음)` 에러를 반환하여 원천 차단
1. 개인정보 보호 (Masking)
- 지자체 담당자라 하더라도 위탁업체 근로자의 민감한 개인정보(주민등록번호 뒷자리, 계좌번호 전체 등)는 필요 이상으로 노출되지 않도록 **마스킹(Masking) 처리**하여 보여주고, 다운로드 파일에도 마스킹된 상태로 다운로드되도록 설계
