# 📋 지자체 모니터링 확장 개발 규격서 (v1.0)

> **목표**: 위탁업체 ERP 데이터를 지자체가 실시간으로 모니터링·검증할 수 있도록 확장. 지자체 신뢰 확보를 통한 영업·계약 우위 확보.
> **참조**: Gemini 제안서 v1.1, 현재 슈퍼관리자 콘솔 ([_super-admin-client.tsx](../../app/(admin)/super-admin/_super-admin-client.tsx))
> **상태**: 검토 단계 (Plan 전)
> **우선순위**: 🔥 High (지자체 영업 차별화 핵심)
> **작성일**: 2026-04-27

---

## 1. 목적

| 항목 | 내용 |
|---|---|
| Why | 지자체가 직접 모니터링할 수 없으면 위탁업체 신뢰 확보 어려움 → 영업 진입 장벽 |
| Who | (1차) 위탁업체 슈퍼관리자, (2차) 지자체 담당 공무원, (3차) 시민 |
| What | 권한 매트릭스 + 증빙 데이터 노출 + 18종 표준 보고서 + 출력 양식 빌더 |
| Success | 지자체 1곳 시범 도입 → 분기 자체 평가 자동화로 담당 공무원 업무 50% 절감 |

## 5. 출력 양식 빌더 — 옵션 비교

| 옵션 | 공수 | 장점 | 단점 |
|---|---|---|---|
| A. 풀 캔버스 빌더 (Gemini 제안) | 4~6주 | GrapesJS·`react-grid-layout` 자유도 최고 | 1인 유지보수 부담 큼 |
| **B. JSON 템플릿 + 미리보기 (권장)** | **1~2주** | **ExcelJS/Puppeteer로 PDF·XLSX 생성. 기존 `reports` 모듈 확장** | **자유도 중간** |
| C. 표준 N개 + 변수 치환 | 3~5일 | 영업 시작 단계 충분 | 지자체별 커스텀 어려움 |

**추천 경로**: C → B → A 단계적. 시범 도입 직전엔 C, 정식 계약 후 B, 다수 지자체 확보 후 A.

### 5.1 옵션 B 핵심 데이터 모델

```prisma
model ReportTemplate {
  id              BigInt    @id @default(autoincrement())
  contractorId    BigInt    @map("contractor_id")
  municipalityId  BigInt?   @map("municipality_id")  // null = 표준 양식
  code            String    @db.VarChar(20)          // F-01, F-02, ...
  name            String    @db.VarChar(100)
  spec            Json                                 // 헤더·셀·필드 매핑·수식
  outputFormats   String    @default("pdf,xlsx") @db.VarChar(50)
  version         Int       @default(1)
  active          Boolean   @default(true)
  createdAt       DateTime  @default(now()) @map("created_at")
  updatedAt       DateTime  @updatedAt @map("updated_at")

  @@unique([contractorId, municipalityId, code, version])
  @@map("report_templates")
}
```

### 5.2 spec JSON 구조 (예시)

```json
{
  "page": { "size": "A4", "orientation": "portrait" },
  "header": [
    { "type": "text", "value": "{{municipality.name}} 일일 수거 운행 일지" },
    { "type": "field", "source": "logDate" }
  ],
  "table": {
    "source": "VehicleLog",
    "filter": { "logDate": "{{date}}" },
    "columns": [
      { "label": "차량번호", "field": "vehicle.plateNumber" },
      { "label": "운전자", "field": "driver.name" },
      { "label": "노선", "field": "zone.name" },
      { "label": "수거(kg)", "field": "wasteWeightKg", "agg": "sum" }
    ]
  },
  "footer": [
    { "type": "signature", "label": "현장책임자" },
    { "type": "signature", "label": "지자체 담당자" }
  ]
}
```
