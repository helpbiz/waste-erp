# 📋 신청 페이지 (`/apply`) 기능 명세서

> **목표**: 위탁대행 업체가 인증 없이 직접 서비스 도입을 신청할 수 있는 공개 페이지 제공.
> **참조**: 서비스_개설_절차.md 부록 C "서비스 신청서 양식"
> **상태**: 명세 단계 (구현 전)
> **우선순위**: 🔥 High (영업 자동화 첫 단계)

---

## 1. 목적

| 항목 | 내용 |
|---|---|
| Why | 영업 매니저 1대1 응대 → **자동 유입 + 검토 큐** |
| Who | 신규 도입 검토 중인 위탁대행 업체 (인증 X) |
| What | 표준 신청서를 웹 폼으로 작성·제출 → 운영팀 슈퍼관리자 콘솔에서 검토 |
| Success | 월 신청 10건+ 자동 접수 / 영업 응답 시간 1일 → 2시간 단축 |

## 4. DB 스키마 추가

```prisma
model ServiceApplication {
  id             BigInt   @id @default(autoincrement())
  applicationNo  String   @unique @db.VarChar(20)      // APP-2026-XXXX
  
  // Step 1: 회사 정보
  companyName    String   @db.VarChar(100)
  businessNo     String   @db.VarChar(20)
  address        String   @db.VarChar(255)
  ceoName        String   @db.VarChar(50)
  ceoPhone       String   @db.VarChar(20)
  contactName    String   @db.VarChar(50)
  contactRole    String?  @db.VarChar(50)
  contactEmail   String   @db.VarChar(100)
  contactPhone   String   @db.VarChar(20)
  
  // Step 2: 위탁계약
  municipalityName  String   @db.VarChar(100)
  contractStart     DateTime @db.Date
  contractEnd       DateTime @db.Date
  contractType      String   @db.VarChar(20)  // SOLE / JOINT
  
  // Step 3: 운영 규모
  totalEmployees    Int
  drivers           Int
  collectors        Int
  cleaners          Int
  managers          Int
  totalVehicles     Int
  
  // Step 4: 도입 일정
  desiredStartDate  DateTime @db.Date
  desiredPlan       String   @db.VarChar(20)  // STARTER / STANDARD / PRO / ENTERPRISE
  needsMigration    Boolean  @default(false)
  legacySystem      String?  @db.VarChar(100)
  painPoints        String[] // 다중 선택
  
  // Step 5: 첨부 서류 (S3 URL 또는 파일 ID)
  businessRegFile   String?  @db.Text
  contractFile      String?  @db.Text
  ceoIdFile         String?  @db.Text
  orgChartFile      String?  @db.Text
  
  // Step 6: 동의
  privacyConsent    Boolean
  marketingConsent  Boolean  @default(false)
  
  // 처리 상태
  status         ApplicationStatus  @default(SUBMITTED)
  submittedAt    DateTime  @default(now())
  reviewedAt     DateTime?
  reviewedBy     BigInt?
  rejectedReason String?   @db.Text
  convertedToContractorId BigInt?  // 승인 후 생성된 위탁업체 ID
  
  // 감사
  ipAddress      String?   @db.VarChar(45)
  userAgent      String?   @db.Text
  
  createdAt      DateTime  @default(now()) @map("created_at")
  
  @@index([status, submittedAt])
  @@index([businessNo])
  @@map("service_applications")
}

enum ApplicationStatus {
  SUBMITTED         // 신청 접수
  UNDER_REVIEW      // 검토 중
  QUOTED            // 견적 발송 완료
  CONTRACTED        // 계약 체결
  ONBOARDING        // 도입 진행 중
  CONVERTED         // 정식 운영 전환 완료
  REJECTED          // 반려
  EXPIRED           // 30일 미응답 만료
}
```

## 5. API 설계

### 5.1 `POST /api/apply` — 공개 (인증 X)

**미들웨어**: 이 경로는 `PUBLIC_PATHS`에 추가 필요.

```typescript
// app/api/apply/route.ts
const Body = z.object({
  // Step 1
  companyName: z.string().trim().min(2).max(100),
  businessNo: z.string().regex(/^\d{3}-\d{2}-\d{5}$/),
  address: z.string().trim().min(5).max(255),
  ceoName: z.string().trim().min(2).max(50),
  ceoPhone: z.string().regex(/^01[0-9]-?\d{3,4}-?\d{4}$/),
  contactName: z.string().trim().min(2).max(50),
  contactEmail: z.string().email().max(100),
  contactPhone: z.string().regex(/^01[0-9]-?\d{3,4}-?\d{4}$/),
  contactRole: z.string().trim().max(50).optional(),
  
  // Step 2
  municipalityName: z.string().trim().min(2).max(100),
  contractStart: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  contractEnd: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  contractType: z.enum(['SOLE', 'JOINT']),
  
  // Step 3
  totalEmployees: z.number().int().min(1).max(10000),
  drivers: z.number().int().min(0).default(0),
  collectors: z.number().int().min(0).default(0),
  cleaners: z.number().int().min(0).default(0),
  managers: z.number().int().min(0).default(0),
  totalVehicles: z.number().int().min(0).max(1000),
  
  // Step 4
  desiredStartDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  desiredPlan: z.enum(['STARTER', 'STANDARD', 'PRO', 'ENTERPRISE']),
  needsMigration: z.boolean().default(false),
  legacySystem: z.string().trim().max(100).optional(),
  painPoints: z.array(z.string().max(100)).max(10),
  
  // Step 5 — 파일은 별도 업로드 API에서 처리 후 URL 전달
  businessRegFile: z.string().url().optional(),
  contractFile: z.string().url().optional(),
  ceoIdFile: z.string().url().optional(),
  orgChartFile: z.string().url().optional(),
  
  // Step 6
  privacyConsent: z.literal(true),
  marketingConsent: z.boolean().default(false),
});

export async function POST(req: Request) {
  const parsed = Body.safeParse(await req.json().catch(() => null));
  if (!parsed.success) return NextResponse.json({ error: 'invalid_request', issues: parsed.error.flatten().fieldErrors }, { status: 400 });
  
  // 1. 중복 신청 체크 (사업자번호)
  const existing = await prisma.serviceApplication.findFirst({
    where: { businessNo: parsed.data.businessNo, status: { in: ['SUBMITTED', 'UNDER_REVIEW', 'QUOTED'] } },
  });
  if (existing) {
    return NextResponse.json({ error: 'duplicate_application', existingNo: existing.applicationNo }, { status: 409 });
  }
  
  // 2. Rate limit (IP 기준 10분 1건)
  const recent = await prisma.serviceApplication.count({
    where: {
      ipAddress: req.headers.get('x-forwarded-for')?.split(',')[0] ?? null,
      submittedAt: { gte: new Date(Date.now() - 10 * 60 * 1000) },
    },
  });
  if (recent >= 1) return NextResponse.json({ error: 'rate_limit' }, { status: 429 });
  
  // 3. 접수번호 생성
  const year = new Date().getFullYear();
  const seq = await prisma.serviceApplication.count({ where: { applicationNo: { startsWith: `APP-${year}-` } } });
  const applicationNo = `APP-${year}-${String(seq + 1).padStart(4, '0')}`;
  
  const created = await prisma.serviceApplication.create({
    data: {
      ...parsed.data,
      applicationNo,
      contractStart: new Date(parsed.data.contractStart),
      contractEnd: new Date(parsed.data.contractEnd),
      desiredStartDate: new Date(parsed.data.desiredStartDate),
      ipAddress: req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ?? null,
      userAgent: req.headers.get('user-agent')?.slice(0, 500) ?? null,
    },
  });
  
  // 4. 자동 이메일 발송 (담당자) — 향후 구현
  // 5. Slack 알림 (운영팀) — 향후 구현
  
  await prisma.auditLog.create({
    data: {
      action: 'SERVICE_APPLICATION_SUBMIT',
      resourceType: 'service_application',
      resourceId: created.id.toString(),
      ipAddress: req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ?? null,
      metadata: { applicationNo, companyName: parsed.data.companyName, businessNo: parsed.data.businessNo } as object,
    },
  });
  
  return NextResponse.json({
    ok: true,
    applicationNo,
    nextStep: '영업일 기준 1~2일 내에 견적과 함께 연락드리겠습니다.',
  });
}
```

// status 전환:

## 6. 슈퍼관리자 콘솔 통합

`/super-admin`에 신규 탭 **"서비스 신청 검토"** 추가:

```
탭 구성:
[지자체 관리] [지자체 권한] [관할 거래처] [회사정보·차고지] [GIS 설정] [📩 서비스 신청 검토]
                                                                            ↑ 신규
```

**기능**:
- 신청 목록 (상태별 필터)
- 신청서 상세 보기 (모든 필드 + 첨부 파일 다운로드)
- 상태 전환 버튼 (검토 시작 → 견적 발송 → 계약 → 도입 시작 → 정식 전환)
- 자동 변환: CONTRACTED 시 `Municipality` + `Contractor` + `CONTRACTOR_ADMIN` 계정 자동 생성

## 7. 보안 + 안전장치

| 항목 | 조치 |
|---|---|
| Rate limit | IP당 10분 1건, 24시간 5건 |
| 봇 차단 | reCAPTCHA v3 (선택) 또는 hCaptcha |
| 파일 업로드 검증 | 5MB 제한, MIME 타입 검사, 바이러스 스캔(ClamAV 옵션) |
| PII 보호 | 사업자번호·연락처는 암호화 컬럼(향후) |
| 중복 방지 | 동일 사업자번호 + SUBMITTED/UNDER_REVIEW 상태 시 409 |
| audit_log | 모든 신청·상태 변경 기록 |

## 9. 메트릭 (KPI)

| 지표 | 측정 방법 |
|---|---|
| 신청 접수 수 | `COUNT(service_applications) GROUP BY MONTH` |
| 전환율 | CONVERTED / SUBMITTED |
| 평균 검토 시간 | `AVG(reviewedAt - submittedAt)` |
| 평균 도입 시간 | `AVG(convertedAt - contractedAt)` |
| 반려율 | REJECTED / SUBMITTED |
