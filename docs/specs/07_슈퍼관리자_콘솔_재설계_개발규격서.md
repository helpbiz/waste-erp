## 1. Executive Summary

| Perspective | Content |
|---|---|
| **Problem** | 슈퍼관리자 콘솔 5개 탭이 flat list·기본 select·단일 차고지 등 초기 MVP 구조에 머물러 영업 시연 시 임팩트 부족 |

| **Core Value** | 1인 운영자가 1주 만에 시연용 핵심 UX 완성 가능. 큰 작업(Geo-fencing·HWP)은 영업 후로 분리 |

### 2.2 Product Manager 평가

- **Phase 1 7개 항목으로 압축** — `Municipality.region` 이미 존재 + F-02 PDF 인프라 재활용. 1주 내 시연 가능

- **Geo-fencing은 Phase 3로 내림** — 폴리곤 설계 + 이탈 감지 + 모바일 권한까지 단독 2주+. 영업 단계엔 "준비 중" 로드맵 슬라이드가 더 효율적

## 4. 즉시 결정 필요 사항 (PM Open Questions)

영업·운영 일정에 영향을 미치는 3건. 답에 따라 Phase 분배가 변동됩니다.

| # | 질문 | 영향 |
|---|---|---|
| Q1 | **HWP 출력은 포기** | 사실상 구현 불가. Excel + PDF 두 형식만 제공해도 OK  |
| Q2 | **차고지 다중 관리는 영업 후 vs 즉시?** | 즉시면 Phase 1 + 1.5주, 영업 후면 Phase 2로. 시연 시 차고지가 차별화 포인트인지 |
| Q3 | **Excel 출력 라이브러리 도입 OK** | `exceljs` ~150KB, `xlsx` ~400KB. F-02 PDF 작업에서 puppeteer 도입과는 별도 영향 |

## 5. 운영 가드 (Phase 1 진입 시 적용)

직전 모바일 메뉴 재설계 규격서 §3 운영 가드와 동일 원칙 적용:

1. **분기 처리** — 슈퍼관리자 콘솔은 데스크톱 우선 + 모바일은 Phase 2 이후 검토
2. **회귀 방지** — 5개 탭 각각 독립 commit. Tab 3은 회귀 검증만, 코드 변경 X
3. **테스트** — Phase 1 작업마다 Playwright 1280 viewport + 768 viewport (태블릿) 검증
4. **배포** — `feat/super-admin-redesign` 전용 브랜치, staging 1주 후 main 병합
5. **Rollback** — 컴포넌트 단위 분리되어 있어 탭별 revert 가능

## 6. 위험 요소 및 mitigation

| # | 위험 | 영향 | 대응 |
|---|---|:---:|---|
| 1 | Tab 5 차고지 1:1 → 1:N 마이그레이션 시 `/live-vehicles` 추천경로 회귀 | High | Phase 2로 분리. 마이그레이션 시 `getDefaultGarage()` 헬퍼 신설로 호환성 유지 |
| 2 | HWP 출력 요구가 영업 단계에서 필수로 굳어지면 외부 변환 서비스 비용 발생 | Medium | Q1 즉시 협의 — Excel + PDF로 충분 |
| 3 | xlsx 라이브러리 추가로 첫 페이지 JS 사이즈 증가 (현재 87.5kB) | Low | `await import()` dynamic import로 출력 액션 시점에만 로드 |
| 4 | Tab 2의 region 필드가 NULL인 지자체 처리 | Low | "미분류" 그룹 slot 추가, region NULL 일괄 채우기 마이그레이션 별도 |
| 5 | 신규 화면 코드(ALL_SCREENS 추가)가 기존 저장된 정책 CSV에 자동 반영 안 됨 | Low | 기존 정책 편집 시 누락 코드 미체크 표시 — 사용자 수동 보정 필요 |
| 6 | F-02 Module 2·3·6·7·8 작업과 동시 진행 시 컨텍스트 분산 | Medium | Phase 1만 우선 처리 → F-02 완료 후 Phase 2 진입 |
