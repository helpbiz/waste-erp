## 3. 운영 가드 (사용자 요청 반영)

### 3.1 분기 처리 강화 — 컴포넌트 코드 레벨 분리

**원칙**: 모바일 메뉴와 데스크톱 사이드바를 **별도 컴포넌트**로 분리. 조건부 렌더링은 한 곳에서만.

```
app/(admin)/_admin-shell.tsx        (router/엔트리)
  ├─ <DesktopSidebar />            (md+ 전용 — 기존 SidebarBody 유지)
  └─ <MobileMenu />                 (모바일 전용 — 신규)
       ├─ <MobileMenuTrigger />     (햄버거 우상단)
       ├─ <MobileMenuPanel />       (드로어 컨테이너)
       │    ├─ <MobileMenuHeader /> (그라데이션 + 아바타 + ✕)
       │    ├─ <MobileMenuList />   (7 카테고리)
       │    └─ <MobileMenuFooter /> (내정보 + 로그아웃)
       └─ <MobileSubPanel />        (드릴다운용 — 카테고리별 서브 항목)
```

**이점**:
- 데스크톱 코드 수정 시 모바일 영향 0
- 각 컴포넌트 단독 테스트 가능
- 향후 워커 앱(`/worker/*`)에서 `<MobileMenu />` 일부 재사용 가능

**가드 코드 패턴**:
```tsx
{/* AdminShell — 명시적 분기 */}
<DesktopSidebar className="hidden md:flex" {...props} />
<MobileMenu     className="md:hidden"      {...props} />
```

`isMobile` JS 분기 대신 **CSS 분기**(`hidden md:flex` / `md:hidden`) 채택 — SSR 안정성 + Hydration mismatch 방지.

### 3.2 스타일 가드 — Breakpoint 고정

**Tailwind breakpoint 합의** (현재 [`tailwind.config.ts`](../../tailwind.config.ts) default):
| Breakpoint | 폭 | 용도 |
|---|---|---|
| `sm` | 640px | 모바일 가로 / 작은 태블릿 |
| **`md` (분기 기준)** | **768px** | **모바일 ↔ 데스크톱 경계** |
| `lg` | 1024px | 데스크톱 |

**가드 룰**:
1. 모바일 컴포넌트(`<MobileMenu />`)는 `<768px`에서만 렌더 → `md:hidden` 강제
2. 데스크톱 컴포넌트(`<DesktopSidebar />`)는 `≥768px`에서만 렌더 → `hidden md:flex`
3. 모바일 전용 스타일은 명확히 격리: `tailwind.config.ts`에서 모바일 전용 색·spacing 토큰 추가 시 `mobile-` prefix 사용 (선택)
4. 데스크톱 전용 hover 효과는 `md:hover:` 또는 `@media (hover: hover)` 가드
5. 인쇄 모드 영향 차단: `print:hidden` 모바일 메뉴에 적용

### 3.3 테스트 케이스 분리

**Playwright 프로젝트 분리** (이미 [`playwright.config.ts`](../../playwright.config.ts) 셋업됨):

| Project | viewport | 테스트 범위 |
|---|---|---|
| `375-iPhone-SE` | 375×667 | **모바일 메뉴 신규 spec** |
| `393-Pixel7` | 393×851 | **모바일 메뉴 신규 spec** |
| `360-GalaxyS` | 360×800 | **모바일 메뉴 신규 spec** |
| `768-iPad` | 768×1024 | 경계 — 데스크톱 사이드바 확인 |
| `1280-Desktop` (신규 추가 권장) | 1280×800 | **데스크톱 회귀 — 사이드바 변화 0** |

**테스트 파일**:
```
e2e/
  mobile-menu/
    open-close.spec.ts        # 햄버거 우상단 → 드로어 우→좌 슬라이드
    drilldown.spec.ts          # 업무·안전보건·기초자료관리 카테고리 진입/뒤로가기
    direct-navigation.spec.ts  # 1항목 카테고리 즉시 이동
    rbac-visibility.spec.ts    # 시스템관리 SUPER_ADMIN만 노출
    footer-actions.spec.ts     # 내정보·로그아웃 동작
  desktop-regression/
    sidebar-unchanged.spec.ts  # 1280 viewport 사이드바 시각 회귀 (toHaveScreenshot)
```

**자동 회귀 룰**:
- PR마다 모든 5개 viewport 실행
- `toHaveScreenshot` threshold 0.2, maxDiffPixelRatio 0.01 (이미 config에 설정됨)
- 데스크톱 회귀 1px 변화 시 PR 차단

### 3.4 배포 파이프라인 관리

**브랜치 전략**:
```
main
  ├─ feat/mobile-menu-redesign      # 본 작업 전용 브랜치
  │    ├─ Phase 1: 분기 분리 (회귀 0 검증)
  │    ├─ Phase 2: 모바일 메뉴 시각 반영
  │    ├─ Phase 3: 드릴다운
  │    └─ Phase 4: 푸터 + 권한 분기
  └─ (다른 작업) feat/f02-*           # F-02 작업과 독립
```

**CI 게이트**:
1. **TypeScript**: `npx tsc --noEmit` — 0 에러 강제
2. **Lint**: ESLint 설정 후 (현재 미구성, 본 작업에서 같이 셋업 권장)
3. **Mobile E2E**: `npx playwright test --project=375-iPhone-SE,393-Pixel7,360-GalaxyS`
4. **Desktop Regression**: `npx playwright test --project=768-iPad,1280-Desktop`
5. **Visual Diff Approval**: 데스크톱 사이드바 스크린샷 1px 변화도 수동 승인 필요
6. **Build**: `npm run build` — production 빌드 성공

**Staging 배포**:
- `feat/mobile-menu-redesign` → staging 환경 배포 → 1주 모니터링
- 모바일 5명 + 데스크톱 5명 사용자 베타 테스트 후 main 병합

**Rollback 전략**:
- 모바일 컴포넌트만 분리되어 있으므로 `<MobileMenu />` 한 컴포넌트만 이전 버전으로 swap 가능
- 운영 환경에서는 git revert 단일 커밋으로 빠른 복구

## 5. 위험 요소 및 mitigation

| 위험 | 영향 | 대응 |
|---|---|---|
| 데스크톱 사이드바 회귀 | High | Visual Regression 1280 viewport baseline 등록, 1px 변화 PR 차단 |
| 드릴다운 패널 깊이 혼란 | Medium | 서브 패널 1단계 한정. 더 깊어지면 풀 페이지로 이동 (`/users` 등) |
| 햄버거 위치 변경 학습 비용 | Low | 모바일 onboarding 토스트 1회 (선택) |
| 드로어 슬라이드 방향 변경 | Low | Tailwind keyframe 미러 — `slide-in-right` 별도 정의 |
| `/payroll` 위치 모호 | Low | "업무" 그룹 잠정 배치 → 사용자 피드백 후 재배치 |
| 워커 앱(`/worker/*`)과 메뉴 일관성 | Medium | 본 작업은 admin shell만. worker 메뉴는 별도 작업 단위 |
| Hydration mismatch | Medium | `isMobile` JS 분기 회피 — CSS 분기(`md:hidden`/`hidden md:flex`) 강제 |
