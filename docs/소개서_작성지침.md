# 서비스 소개서 작성 지침서

> CleanERP 영업·홍보용 서비스 소개서(`/intro`)를 수정·확장할 때 따라야 할 가이드. 영업 미팅 상대에 따라 지자체·위탁업체 예시 데이터를 갈아끼우거나, 새 슬라이드를 추가하거나, 메시지 톤을 다듬을 때 이 문서를 참고하세요.
> **핵심 원칙: 디자인은 그대로, 데이터·카피만 한 곳에서 변경.**

**대상 독자**: 소개서 카피·시각 자산을 관리하는 영업·마케팅·운영자
**최종 갱신**: 2026-05-02
**관련 문서**: [매뉴얼_작성지침.md](./매뉴얼_작성지침.md)

## 1. 소개서란

CleanERP 서비스 소개서는 **영업 미팅·도입 제안·홈페이지 다운로드용** 27 슬라이드 deck입니다. 무사퇴근 PDF 디자인 시스템을 차용해 helpbiz 브랜드 톤으로 재구성했습니다.

### 1.1 라우트 + 출력

| 형태 | 위치 | 용도 |
|---|---|---|
| 웹 시연 | `/intro` (https://wci.helpbiz.kr/intro) | 영업 미팅 노트북 시연 |
| PDF 출력 | 우측 상단 [PDF 저장] 버튼 → 인쇄 → "PDF로 저장" | 이메일 첨부·인쇄 배포 |

### 1.2 디자인 톤

| 항목 | 결정 |
|---|---|
| 표지 카피 방향 | **듀얼 포지셔닝** — "운영 효율 + 산업안전보건" 두 축 병치 |
| 슬라이드 비율 | 16:9 (1280×720 인쇄 캔버스) |
| 다크/라이트 리듬 | 표지·챕터 디바이더·연락처·Thank You = 다크 / 본문 = 라이트 |
| 액센트 컬러 | 시안 `#06b6d4` (helpbiz 브랜드 teal에서 파생) |
| 다크 배경 | 네이비→딥블루 그라디언트 + 시안 스폿라이트 |
| 폰트 | Pretendard Variable (한글 가독성) |
| 단위 | `cqw` 컨테이너 쿼리 — 슬라이드 폭 비례 자동 스케일 |

### 1.3 슬라이드 구성

| 챕터 | 슬라이드 | 패턴 |
|---|---|---|
| **표지** | 01 | dark cover (헤드라인 + 브랜드) |
| **목차** | 02 | dark TOC (6 챕터 + 시작 페이지) |
| **개발 배경** | 03 디바이더 / 04 3대 통증 / 05 통계 / 06 Before-After | dark + light 본문 3종 |
| **CleanERP 소개** | 07 디바이더 / 08 운영사·도메인 / 09 Role 트리 / 10 멀티테넌시 | dark + light 본문 3종 |
| **주요 기능** | 11 디바이더 / 12 타임라인 / 13~19 기능 7종 / 20 보안 / 21 매트릭스 | dark + light 본문 11종 |
| **도입 절차** | 22 디바이더 / 23 4-스텝 | dark + light 본문 1종 |
| **요금제** | 24 디바이더 / 25 요금표 | dark + light 본문 1종 |

## 2. 파일 구조

```
app/intro/
├── _config.ts              # 가변 데이터 (브랜드·예시 지자체·위탁업체·연락처)
├── _content.ts             # 모든 슬라이드 카피 (표지/챕터/통증/기능/요금 등)
├── _print-button.tsx       # PDF 저장 버튼 (client component)
├── intro.css               # 디자인 시스템 (~900 lines, 슬라이드 패턴 16종)
└── page.tsx                # 27 슬라이드 deck (디자인만, 카피 X)
```

### 2.1 무엇을 만지면 되나?

| 작업 | 만지는 파일 | 영향 범위 |
|---|---|---|
| 회사명·도메인·연락처 변경 | `_config.ts` (BRAND) | 전체 슬라이드 자동 갱신 |
| 예시 지자체·위탁업체 변경 (영업 컨텍스트별) | `_config.ts` (EXAMPLE_*) | 멀티테넌시·민원·실적·매트릭스 슬라이드 자동 갱신 |
| 슬라이드 카피 다듬기 (헤드라인·바디) | `_content.ts` (COPY) | 해당 슬라이드만 |
| 새 슬라이드 추가 | `_content.ts` + `page.tsx` (한 섹션 추가) | 추가된 슬라이드 |
| 요금 단계·금액 변경 | `_content.ts` (COPY.pricing.rows) | 요금제 슬라이드 |
| FAQ·도입 사례 추가 | `_content.ts` + `page.tsx` (선택) | 추가된 영역 |
| 디자인 변경 (색·여백·폰트) | `intro.css` | **전체** |
| 슬라이드 재배치 | `page.tsx` (섹션 순서) | 전체 흐름 |

**가장 흔한 작업: 영업 미팅 상대 바뀔 때 예시 지자체 갈아끼우기.**

## 3. 영업 컨텍스트별 데이터 갈아끼우기 (가장 흔한 작업)

### 3.1 시나리오

송파구청과 미팅이 있는데, 현재 소개서는 용산구청 중심으로 작성되어 있다면?

`_config.ts` **한 파일만** 편집하시면 됩니다.

### 3.2 변경 예시

```tsx
// app/intro/_config.ts

// Before (용산구청 중심)
export const EXAMPLE_MUNI = {
  primary: {
    name: '용산구청',
    district: '용산구',
    neighborhoods: ['이태원동', '한남동', '후암동'] as const,
  },
  secondary: ['송파구청', '밀양시 시설관리공단', ...],
};

export const EXAMPLE_CONTRACTORS = ['용산청소(주)', '송파환경(주)', ...];

// After (송파구청 중심)
export const EXAMPLE_MUNI = {
  primary: {
    name: '송파구청',
    district: '송파구',
    neighborhoods: ['잠실동', '문정동', '오금동'] as const,
  },
  secondary: ['용산구청', '밀양시 시설관리공단', ...],   // primary와 swap
};

export const EXAMPLE_CONTRACTORS = ['송파환경(주)', '용산청소(주)', ...]; // 첫 행이 N 뱃지 받음
```

### 3.3 자동 갱신되는 슬라이드

`_config.ts` 한 곳만 바꾸면:

- **슬라이드 10** (멀티테넌시 다이어그램) — 첫 행 지자체·위탁업체
- **슬라이드 13** (민원관리 mock) — 동(洞) 이름 3개
- **슬라이드 17** (실적 보고서) — "○○구청 월간 보고서"
- **슬라이드 19** (슈퍼관리자 콘솔 mock) — 권한 매트릭스 첫 행

표지·챕터 디바이더·요금표 등은 그대로.

### 3.4 권한 매트릭스 시나리오 변경

```tsx
// app/intro/_config.ts

export const EXAMPLE_PRESETS = [
  { muni: '송파구청',           preset: '표준 프리셋',   scope: '...', dl: '✓' },
  { muni: '용산구청',           preset: '모니터링 전용', scope: '...', dl: '✗' },
  { muni: '밀양시 시설관리공단', preset: '전체 공개',     scope: '...', dl: '✓' },
];
```

→ 슬라이드 19 자동 갱신.

## 4. 카피 수정 (표제·바디 다듬기)

### 4.1 워크플로우

1. `app/intro/_content.ts` 열기
2. 수정할 슬라이드의 키 찾기 — `cover`, `pain`, `stat`, `compare`, `about`, `roles`, `tenancy`, `featureTimeline`, `features.complaints`, ..., `pricing`, `contact`, `thanks`
3. 텍스트만 수정
4. 저장 → 브라우저 새로고침

### 4.2 표지 카피 변경

표지 헤드라인은 강조 단어가 분리되어 있습니다:

```tsx
// _content.ts
cover: {
  tag: '226개 지자체와 위탁업체가 함께 쓰는',
  headline: {
    line1: { pre: '운영은 ',   accent: '자동화', post: '하고,' },
    line2: { pre: '안전은 ',   accent: '시스템', post: '으로 지킵니다.' },
  },
  tagline: '생활폐기물 수집운반업을 위한 운영·안전 통합 ERP',
  footerBrand: BRAND.companyUpper,
  footerYear: '2026',
}
```

→ 렌더 결과: "운영은 **자동화**하고, / 안전은 **시스템**으로 지킵니다." (시안색 강조)

표지 카피 변경 시 **anchor·post 비율을 균형 있게** 유지하세요. accent가 너무 길면 시각 균형 깨짐.

### 4.3 카피 톤 가이드

| 영역 | 톤 |
|---|---|
| 표지 헤드라인 | 짧고 강함. 동사형 종결 (~합니다). 한 라인 최대 12자 권장. |
| 챕터 디바이더 sub | 중간 길이 (60~120자). 챕터의 의미 한 단락. |
| 통증·기능 카드 본문 | 1~2 문장. 100자 내. |
| 요금제 | 숫자 정확히. "VAT 별도" 명시. |
| 연락처 | 격식 (~하세요·~드립니다). |

### 4.4 듀얼 포지셔닝 원칙

소개서 카피는 **운영 효율 + 산업안전보건** 두 축을 병치합니다:

- 한 슬라이드에서 한 메시지만 — 운영이면 운영, 안전이면 안전
- 표지·요약 슬라이드는 두 축 동시 — "운영은 자동화, 안전은 시스템"
- 7개 기능 슬라이드: 민원·근태·차량·실적은 운영 톤 / 안전·NOC는 안전 톤

→ 카피 변경 시 어느 축의 메시지인지 판단하고 어휘 선택.

## 5. 새 슬라이드 추가

### 5.1 가장 흔한 케이스: 도입 사례 추가

```tsx
// _content.ts에 추가
caseStudy: {
  tag: '도입 사례',
  title: '용산구청 — 도입 6개월 만에 민원 처리 시간 38% 단축',
  body: '...',
  metrics: [
    { label: '처리 평균', before: '2.4일', after: '1.5일' },
    { label: '기한초과율', before: '12%', after: '0%' },
  ],
}

// page.tsx 적절한 위치에 섹션 추가
<section className="slide slide-light slide-stat" aria-label="도입 사례">
  <span className="pain-tag">{C.caseStudy.tag}</span>
  <h2 className="pain-title">{C.caseStudy.title}</h2>
  ...
  <div className="slide-num">XX</div>
</section>
```

### 5.2 슬라이드 번호 재정렬

새 슬라이드 추가 시 후속 슬라이드의 `slide-num` 값이 모두 1씩 증가해야 합니다. 한 번에 일괄 변경 권장:

```bash
# 예: 슬라이드 17 추가로 18~28 번호 모두 +1
sed -i 's/slide-num">17/slide-num">18/g' app/intro/page.tsx
sed -i 's/slide-num">18/slide-num">19/g' app/intro/page.tsx
# (역순으로 진행 — 27→28부터 위로)
```

또는 page.tsx에서 직접 수정.

TOC도 함께 업데이트:

```tsx
// _content.ts
toc: {
  items: [
    { num: '01', label: '개발 배경',     en: 'The Why',     page: '04' },
    { num: '02', label: 'CleanERP 소개', en: 'About',       page: '08' },
    { num: '03', label: '주요 기능',     en: 'Features',    page: '13' },  // 기존 12에서 13으로
    ...
  ],
}
```

### 5.3 새 챕터 추가 (도입 사례 챕터 추가 같은 큰 변경)

1. `_content.ts`의 `chapters` 객체에 새 챕터 추가:
   ```tsx
   chapters: {
     ...
     7: { en: 'CHAPTER 07 · CASE STUDY', num: '07', title: '도입 사례', sub: '...' },
     total: '/07',  // 분모 갱신
   }
   ```

2. 모든 챕터의 `total` 값 갱신: `/06` → `/07`

3. page.tsx에 새 챕터 디바이더 + 본문 슬라이드 추가:
   ```tsx
   <ChapterSlide n={7} num="XX" />
   <section ...본문 슬라이드>
   ```

4. TOC items 배열에 새 챕터 추가

5. 인쇄·렌더 검증

### 6.2 cqw 단위 사용 규칙

모든 슬라이드 내부 타이포·간격은 **`cqw` (container query width) 단위** 사용:

- `1cqw` = 슬라이드 폭의 1%
- 슬라이드 폭 1280px → 1cqw = 12.8px
- 슬라이드가 축소돼도 비율 자동 유지

```css
.slide-headline { font-size: 5cqw; }   /* 슬라이드 폭에 비례 */
.slide-padding   { padding: 6cqw; }
```

→ **px·rem 직접 사용 금지** (반응형 깨짐).

### 6.3 다크/라이트 리듬

- 다크 → 라이트 → 라이트 → 다크 식의 **2~4 슬라이드 단위 교차** 권장
- 챕터 디바이더(다크) 직후 본문(라이트) → 시각적 휴식
- 다크 슬라이드 연속 X (단, 표지+목차 같은 의도된 경우 제외)

### 6.4 인쇄 (PDF) 안전

- `@page { size: 1280px 720px; margin: 0; }` 고정
- 모든 색상은 `print-color-adjust: exact`로 보존
- 슬라이드는 1페이지에 1개씩 (`page-break-after: always`)

→ `intro.css`의 `@media print` 섹션 수정 시 신중히.

## 7. PDF 출력

### 7.1 사용자 워크플로우

1. `/intro` 진입 (브라우저)
2. 우측 상단 [PDF 저장] 버튼 클릭
3. 인쇄 다이얼로그에서 다음 옵션 선택:
   - 대상: "PDF로 저장"
   - 페이지 크기: 사용자 정의 또는 그대로
   - 여백: 없음 / 0
   - 컬러: 컬러 (기본값)
   - 배경 그래픽: **활성화** (체크) — 그라디언트·강조 색상 보존을 위해 필수

### 7.2 PDF 검증 체크리스트

- [ ] 표지 그라디언트 보존됨
- [ ] 챕터 디바이더 다크 배경 보존됨
- [ ] 시안 액센트 색상 보존됨
- [ ] 표지·연락처에 그림자·블러 효과 보존됨
- [ ] 페이지 잘림 없이 27 페이지 (또는 슬라이드 수만큼)
- [ ] 텍스트 가독성 (8pt 이상 유지)

문제 시 [intro.css](../app/intro/intro.css) `@media print` 섹션 점검.

| page.tsx에 카피 직접 인라인 | 카피 수정 시 디자인 파일까지 건드려야 함 | _content.ts에 정의 후 import |
| 인쇄 시 "배경 그래픽" 체크 안 함 | 그라디언트·다크 배경 사라짐 | 인쇄 다이얼로그에서 옵션 활성화 |
| 슬라이드 추가 후 TOC 갱신 안 함 | 목차와 실제 페이지 번호 불일치 | TOC items 함께 수정 |
| 표지 accent 단어 너무 길게 | 시각 균형 깨짐 | 2~3자 권장 ("자동화"·"시스템" 등) |
