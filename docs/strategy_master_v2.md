## 1. 포지셔닝 재정의 — 무엇을 파는가

### 잘못된 포지셔닝 (기존)
> "원가산정 자동화 소프트웨어"

### 올바른 포지셔닝 (수정)
> **"10년 현장 검증 역산법을 탑재한 전국 표준 원가산정 플랫폼"**

차이가 중요한 이유:
- 소프트웨어 → 가격 경쟁, 모방 가능
- 검증된 방법론 탑재 플랫폼 → 데이터 네트워크 효과, 모방 불가

**경쟁자가 복제할 수 없는 3중 해자(Moat)**:

```
① 10년 보정계수 DB
   차량 공칭 용량 vs 실적재율 (5톤차 실적 3.2~4.1톤)
   지역 유형별 계수 (도심/농촌/혼합)
   감사원 지적 사례 기반 안전 계수
   → 이 데이터를 얻으려면 수년의 현장 경험 필요

② gis 폐기물 예측 모델
   용산 181만 GPS 포인트 + Ridge R²=0.85
   전국 212개 시군구 발생량 예측
   → 역산 엔진의 입력값을 자동화하는 유일한 시스템

③ wci 실측 검증 루프
   실제 운영 중인 업체의 차량·인원·실적이 실시간 연동
   예측값 vs 실측값 ±15% 자동 검증
   → 설계금액의 정확도를 사후에 증명 가능한 유일한 구조
```

## 2. 도메인 구조 재확인 — 프로세스 3단계

```
[1단계: 설계]           [2단계: 입찰·계약]        [3단계: 사후정산]
                                                    
지자체                   업체                       업체 → 지자체
    ↓                       ↓                           ↓
기후환경에너지부          설계금액 역산 추정          실적 증빙 제출
지침 기반                 낙찰율로 투찰가 결정        3항목 정산
설계금액 산출             (87~92% 구간 경쟁)         ① 직접노무비
    ↓                       ↓                       ② 산업안전보건관리비
입찰공고                 낙찰·계약                   ③ 감가상각비
                                                    → 차액 정산
```

**역산법의 위치**:
```
전국 폐기물 수거량 데이터
    ↓ (gis Ridge 예측 or 공공데이터)
수거량 ÷ 차량 적재량 ÷ 운행횟수 = 필요 차량 대수
    ↓
차량 대수 × 차량당 인원(직종별) = 필요 인원
    ↓
표준품셈 자동 적용
    ↓
설계금액 = 직접노무비 + 차량비 + 간접비 + 일반관리비 + 이윤
```

**핵심 통찰**: "나머지는 표준품셈으로 계산하는 과정이라 어렵지 않다"
→ 어려운 것은 차량·인원 산정의 보정계수, 이것이 10년 경험의 본질

## 5. 원가산정(cost) 프로그램 핵심 설계

### 5.1 역산 엔진 계산 구조

**차량 산정 공식** (차종별 분리 필요):

| 차종 | 적재용량 | 1일 운행횟수 | 차량당 인원 | 비고 |
|------|---------|------------|-----------|------|
| 소형 압축차 (<5톤) | 2.5톤 | 2~3회 | 운전원1+미화원2 | 골목길 특화 |
| 중형 압축차 (5~9톤) | 5.0톤 | 1~2회 | 운전원1+미화원2 | 표준형 |
| 대형 압축차 (9톤+) | 9.0톤 | 1~2회 | 운전원1+미화원2 | 대로변 |
| 덤프트럭 | 8.0톤 | 2~3회 | 운전원1+미화원1 | 대형폐기물 |

```
필요 차량 대수
  = ceil(월간 수거량(톤) ÷ 차량 적재용량 ÷ 월간 가동일수 ÷ 1일 운행횟수)
  × 지역 보정계수 (경사도·골목 비율·계절)

필요 인원 = Σ(직종별 인원) × 교대 수 × 휴가버퍼(1.15)
           + 현장반장(차량5대당 1명) + 관리직(20대당 1명)
```

**보정계수 DB (10년 경험의 핵심 IP)**:

```
보정계수 항목       범위        결정 기준
─────────────────────────────────────────
경사도 계수         1.0~1.15    GIS 경사 데이터 + 현장 확인
골목길 비율 계수    1.0~1.20    GIS 도로폭 데이터
계절 변동 계수      0.95~1.10   과거 발생량 월별 패턴
압축효율 계수       1.0~1.30    차종·폐기물 성상별 실적
```

### 5.2 표준품셈 DB 구조 (연도별 자동 갱신)

```
cost_standard_items 테이블
  year              연도 (매년 갱신)
  item_code         품셈 코드 (지침 코드)
  category          'labor' | 'vehicle' | 'indirect'
  vehicle_type      차종 (차량비 항목에 해당)
  job_type          직종 (노무비 항목에 해당)
  unit_price        단가 (원)
  source            근거 (지침명 + 조항)
  effective_from    적용 시작일
  effective_to      적용 종료일 (NULL = 현행)
```

**지침 갱신 프로세스 (연 1회)**:
```
기후환경에너지부 고시 발표
    ↓
관리자가 신규 연도 파라미터만 업데이트
    ↓
전국 모든 지자체 계산 자동 반영
    ↓
"2026년 지침 반영 완료" 공지
```

### 5.3 자동화 vs 전문가 판단 분리

| 영역 | 처리 방식 | 이유 |
|------|---------|------|
| 표준품셈 계산 (직·간접비, 요율 적용) | **완전 자동** | 규정값, 오차 없음 |
| gis 수거량 예측 연동 | **완전 자동** | API 연동 |
| wci 실적 대비 검증 (±15%) | **완전 자동** | 기준 명확 |
| 지역 보정계수 (경사·골목) | **자동 추천 + 전문가 확인** | GIS 계산 + 현장 판단 |
| 차종 믹스 결정 | **유사 지역 참조 + 전문가 선택** | 지역 특성 판단 |
| 비정형 지역 처리 | **전문가 수동** | 재개발·특수 지역 |

## 6. 전략 4가지 + 전술

### 전략 ① 지자체를 무료 관문으로, 업체에서 수익 창출

**전술**:
1. 지자체 담당자에게 **무료 설계금액 산출 도구** 제공
2. 지자체가 입찰공고 등록 시 → 참가 업체에 cost 플랫폼 노출
3. 업체는 낙찰율 시뮬레이터·사후정산 자동화 유료화
4. "지자체가 쓰는 시스템" = 신뢰 + 홍보 자동화

---

### 전략 ② 지침 파라미터 DB = 플랫폼의 핵심 해자

**전술**:
1. 기후환경에너지부 고시 발표 즉시 파라미터 업데이트 (연 1회)
2. 5년치 지침 이력 보관 → 과거 설계금액 재현 가능 (감사 대응)
3. "올해 지침 자동 반영" = 경쟁 불가 서비스
4. 장기적으로 지침 개정 작업에 의견 제출 → 플랫폼이 사실상 표준화

---

### 전략 ③ wci 운영 데이터 = 사후정산 증빙 자동화 무기

**전술**:
1. wci 단독 사용자 → 사후정산 증빙 수작업 (고통 지속)
2. wci + cost 패키지 → 사후정산 자동화 활성화 (고통 해소)
3. 패키지 번들 할인 → 단독 사용자의 묶음 가입 유도
4. "wci를 써야 사후정산이 자동화된다" = wci 이탈 방지 + cost 가입 유인 동시 달성

---

### 전략 ④ G2B 낙찰율 DB = 업체 킬러 기능

**전술**:
1. gis의 기존 G2B 수집 기능 → 폐기물 입찰 낙찰 결과 자동 수집 확장
2. 지역별·규모별·연도별 낙찰율 분포 통계 구축
3. 이 데이터는 나라장터 공개 데이터지만 **분석·시각화 형태는 독점**
4. 입찰 1건 수익이 수억원인 업체에게 월 10~30만원은 허들이 아님

## 9. 레퍼런스 확보 전략 (가장 중요)

**첫 3개 지자체 선정 기준**:

```
1순위: 기존 용역 수행 지자체 중 담당자 신뢰 관계 있는 곳
2순위: 인구 10~30만 규모 (너무 크지도 작지도 않음)
3순위: 최근 원가산정 감사 지적 받은 곳 (고통 포인트 명확)
```

**전환 제안**:
- "기존 용역료와 동일 금액 (2천만원)으로 1년 무제한 사용 + 보고서 제공"
- 지자체 입장: 3년 주기 2천만원 → 연간 구독으로 예산 분산 + 지침 갱신 자동화

**학술 신뢰도 병행 구축**:
- 현재 준비 중인 Ridge 논문 (KSEE 투고) + 역산법 방법론 논문 추가
- "학술 검증된 방법론을 탑재한 플랫폼" = B2G 신뢰의 핵심

  역산법은 어렵지 않다 (표준품셈으로 해결)
  어려운 것은 차량·인원 산정의 보정계수 = 10년 경험

세 프로그램의 분업:
  gis  →  수거량 예측 (역산의 입력값 자동화)
  cost →  역산 엔진 + 설계금액 + 낙찰 전략 + 사후정산
  wci  →  운영 실적 (사후정산 증빙 + 예측 검증)

경쟁자가 복제 불가능한 것:
  10년 보정계수 DB + gis 예측 연동 + wci 실측 검증
  = 전국에서 유일한 3중 데이터 파이프라인

수익 전환:
  현재  →  1인 × 연 5~10건 × 2천만원 = 1.5억원
  3년차 →  플랫폼 × 100+개 고객 동시 = 2.4억원+
  본질  →  같은 방법론으로 5배 이상 레버리지

첫 행동:
  기존 고객 지자체 3개 → 베타 전환 → 레퍼런스 확보
  역산 엔진 코어 개발 (fleet_simulator.py 재활용)
  학술 논문 발표 병행 → B2G 신뢰 구축
