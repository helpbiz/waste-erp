## 0. 분석 배경 (왜 다시 그리는가)

CleanERP는 4-role × 다수 페이지 구조의 운영 ERP를 PWA로 배포 중이다. Wave 1~4에서 모바일 네비게이션을 점진 개선했지만 다음 누적 실패가 관찰됐다:

1. **고령 미화원이 로그아웃 버튼을 못 찾는다** (12px / text-slate-200 / 호버 underline / 16px 터치타겟) — 시인성·접근성·터치타겟 3중 위반.
2. **PWA임에도 핀치 줌이 자유로워 "앱 같지 않다"** (`viewport`에 `maximum-scale`/`user-scalable` 미설정).
3. **role × 페이지마다 셸/네비/패턴이 달라** 인지부하가 누적된다 (관리자 셸, 워커 셸, 시민 신고 셸이 각자 다른 룰을 적용).

부분 패치(Wave 1~4)로 표면만 손본 결과, 진짜 페르소나(50~60대 시니어 환경미화원)에게는 여전히 "안 보이고, 끌리고, 매번 다른 앱"이다. 본 Discovery는 **처음부터 다시 그린다**는 사고로 5-Step Chain과 OST를 통해 기회를 재정렬한다.

### 2.1 기회 1: "안 보인다" — 가독성/대비 결함

**증거**: 사용자 직접 보고. 12px / text-slate-200 / 16px 터치 = 3중 위반. 환경미화원 평균 50~60대 노안.

**솔루션**:
- **1.1 WCAG 2.1 AAA 디자인 토큰 체계** (대비 7:1, 본문 18px+, 터치 44×44px+) — 색·폰트·간격을 토큰화하여 한 곳에서 통제
- **1.2 야외 가독성 모드**: 햇빛 모드(고대비) / 야간 모드(저눈부심) — 미화원 새벽/야간 근무 대응. OS 다크모드 + 명시적 토글 병행

**검증 지표**: 시니어 5초 가독성 5/5, 야외 환경 정보 발견 시간 30%↓

### 2.2 기회 2: "앱 같지 않다" — PWA 신뢰도

**증거**: 사용자 직접 보고 (핀치 줌 자유로 화면 끌림).

**솔루션**:
- **2.1 viewport 잠금 + safe-area-inset**: `maximum-scale=1, user-scalable=yes` 동시 (잠긴 느낌 + 줌 접근성). iOS notch / Android nav bar 대응
- **2.2 PWA 셸 강화**: status bar 색상 통일, 스플래시 스크린, 홈화면 설치 유도 onboarding 카드, "앱 모드(standalone)" 감지 시 미세 차별화

**검증 지표**: 잠금 빌드 NPS +1점, "앱 같다" 정성 응답

### 2.3 기회 3: "매 페이지 패턴이 달라" — 인지부하

**증거**: 사용자 직접 보고. 4-role × 다수 페이지의 셸/네비 비일관.

**솔루션**:
- **3.1 4-role 통합 셸**: 단 하나의 `<AppShell>` 컴포넌트가 role을 받아 헤더(타이틀·아바타), 하단 탭(role별 5탭), FAB(주요 CTA)을 렌더. 모든 라우트가 셸 안에서 산다.
- **3.2 단일 디자인 시스템 토큰**: Tailwind theme + (선택)shadcn-style primitives. 색·간격·라운드·그림자·모션 — 한 토큰 정의를 공유.

**검증 지표**: role 겸직 사용자 SEQ ≤ 3 (5점 척도, 낮을수록 쉬움), 신규 사용자 두 번째 페이지 학습 시간 50%↓

### 2.4 기회 4: "내 role에 맞는 메뉴만 보이지 않음"

**증거**: 사용자 명세 (RAPID 직책일 때만 '최적경로계산' 표출). 기획자 우려.

**솔루션**:
- **4.1 RBAC 기반 메뉴 분기**: 중앙 메뉴 정의 (`menu.config.ts`)에 role × subRole 매트릭스. 라우트 가드 + UI 가드 동시.
- **4.2 기동반(RAPID) 전용 라우팅**: '최적경로계산' 메뉴를 RAPID에게만 노출 + 딥링크 보호 (직접 URL 접근 시 차단).

**검증 지표**: 일반 worker가 RAPID 전용 메뉴를 보거나 누르는 비율 0%, RAPID는 1탭으로 도달
