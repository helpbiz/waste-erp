## 3. 시장 사이징 (TAM/SAM/SOM)

### 3.1 Top-Down 추정

| 단계 | 정의 | 수량/금액 | 근거 |
|------|------|----------|------|
| **TAM** | 한국 전국 지자체 생활폐기물 수집운반 ERP 시장 (위탁대행사 운영 ERP) | 약 226개 시군구 × 평균 위탁사 2~5곳 ≈ **500~1,000 위탁사** | 한국 행정구역 + 환경부 자료 [추정] |
| 환경미화원 인구 | 위 위탁사가 고용한 미화원 | **약 4~6만 명** | 환경부/통계청 환경미화원 통계 [추정] |
| **SAM** | PWA/모바일 ERP 도입 의지 + 예산 보유 위탁사 (디지털화 추진 지자체) | TAM의 30~50% ≈ **150~500 위탁사** | 공공 디지털 전환 정책, 발주 트렌드 [추정] |
| 미화원 수 (SAM) | 위 SAM 위탁사 미화원 | **2~3만 명** | 위와 비례 |
| **SOM** | CleanERP가 향후 3년 내 도달 가능 (현 운영 + 신규 도입) | SAM의 5~15% ≈ **20~70 위탁사** | 현재 wci.helpbiz.kr 운영 + 한국 시장 진입속도 [가설] |
| 미화원 수 (SOM) | 위 SOM 위탁사 미화원 | **1,000~5,000 명** | DAU/MAU 측정의 분모 |

### 3.2 Bottom-Up 검증

> 1개 베타 위탁사에서 "30명 미화원 × 자력 사용 100% × DAU/MAU 0.7"을 달성했을 때, 인접 위탁사 5곳에 레퍼런스로 확산 → 첫 12개월 5~10 위탁사, 18~24개월 20~30 위탁사 SOM 진입 — 위 Top-Down SOM의 하단과 일치.

### 3.3 본 이니셔티브의 사이징 의의

본 이니셔티브는 **신규 매출 창출**이 아니라 **SOM 진입 마찰 제거**다:

- 시니어 자력 사용률 100% → 신규 위탁사 도입 결정의 핵심 의문 ("우리 직원들이 쓸 수 있을까?")을 제거
- WCAG 2.1 AAA → 공공 발주 RFP 가점 → 발주 통과율 상승
- 4-role 통합 셸 → 위탁사 도입 후 확장(직원 추가, manager 등록, muni 연계)이 한 번에

## 5. Job Stories (JTBD 형식)

> "**When** [상황], **I want to** [동기], **So I can** [기대 결과]." (Alan Klement)

### 5.1 Worker (P1, P2)

- **JS-W1**: When 매일 새벽 출근하여 PWA를 처음 열 때, I want to 출근 체크인을 1탭으로 끝내고 싶다, So I can 동료한테 묻지 않고 바로 작업으로 넘어간다.
- **JS-W2**: When 안전점검을 시작할 때, I want to 큰 체크박스(28px+)와 한 번에 한 질문씩 만나고 싶다, So I can 노안에도 빠짐없이 점검한다.
- **JS-W3**: When 장갑 끼고 폰을 잡을 때, I want to 모든 핵심 버튼이 풀-너비/하단에 있길, So I can 손가락이 빗나가지 않는다.
- **JS-W4 (RAPID)**: When 출근 직후 신고 큐를 보고 경로를 짤 때, I want to '최적경로계산'이 첫 화면 1탭에 있길, So I can 일반 미화원 메뉴 사이에서 헤매지 않는다.
- **JS-W5**: When 작업 종료 후 폰을 끌 때, I want to 로그아웃 버튼이 큰 글씨로 분명히 보이길, So I can 헤매지 않고 다음 사람에게 폰을 넘긴다 (지자체 지급폰 공유 시).

### 5.2 Company (P3)

- **JS-C1**: When 차로 이동 중일 때, I want to 오늘의 운영 KPI 카드 3장을 한 화면에 보고 싶다, So I can 사무실 가지 않고 운영을 통제한다.
- **JS-C2**: When 결재 알림이 도착할 때, I want to 1탭으로 카드를 열고 1탭으로 승인하고 싶다, So I can 직원 대기 시간을 없앤다.

### 5.3 Manager (M)

- **JS-M1**: When 사고/민원이 발생할 때, I want to 푸시 알림 + 큰 카드로 즉시 인지하고 싶다, So I can escalation이 묻히지 않게 한다.

### 5.4 Muni (P4)

- **JS-Mu1**: When 시민 민원이 발생할 때, I want to 위탁사 배정→처리 흐름을 모바일에서 한 화면에 추적하고 싶다, So I can 감사 책임이 깨지지 않는다.
