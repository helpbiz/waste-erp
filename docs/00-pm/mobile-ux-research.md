## 1. 한국 시장 앱 인터페이스 패턴 분석

### 1.1 분석 대상 앱 개요

아래 6개 앱을 현장 맥락(배달·운전·근태·금융)으로 분류하여 인터페이스 패턴을 도출했다.
공개 앱스토어 정보, 사용자 리뷰, 테크 미디어 분석 자료를 근거로 삼았다.

#### [앱 1] 토스 (Toss) — 금융 앱, 초심자 친화 UX 벤치마크

**네비게이션 패턴**

```
┌────────────────────────────┐
│  ← 토스                    │  ← 상단 앱바 (얕은 뎁스)
├────────────────────────────┤
│                            │
│   [카드형 홈 대시보드]     │  ← 스크롤 가능 카드 목록
│   잔액: ₩1,250,000         │
│   [송금] [충전] [결제]     │  ← 주요 CTA 3개 그룹
│                            │
├────────────────────────────┤
│ 홈 | 혜택 | 전체 | 나의   │  ← 하단 탭바 4개
└────────────────────────────┘
```

| 분석 차원 | 토스의 패턴 |
|-----------|------------|
| 네비게이션 | 하단 탭바 4개 (항상 고정 표시) |
| CTA 위치 | 화면 중상단 카드 내부, 엄지 도달 가능 영역 |
| 모달 vs Sheet | 토스 고유 패턴: Alert(중앙·불투명도 40%) vs Sheet(하단·불투명도 20%) 명확 구분 |
| 다크모드 | 자동 지원, OS 설정 연동 |
| 글자 크기 | 잔액 표시 28sp, 본문 16sp, 레이블 12sp |
| 카드 스타일 | 흰 배경·미세 그림자·8dp 라운드 코너 |

**우리 제품 차용 패턴**
1. **Sheet vs Alert 이중 모달 전략**: 긴급 알림(안전사고 신고 등)은 중앙 Alert, 맥락적 입력(민원 응답 내용 입력)은 하단 Sheet로 구분 → 사용자가 상황을 즉각 판단 가능
2. **홈 카드 대시보드**: 오늘의 출퇴근 상태·안전점검 완료 여부·미확인 민원을 카드 3개로 첫 화면 요약

*근거: [토스 모달 설계 분석 — Medium by Beomsu Byun](https://medium.com/@beomsu/%ED%86%A0%EC%8A%A4%EB%8A%94-%EB%AA%A8%EB%8B%AC%EC%9D%84-%EC%96%B8%EC%A0%9C-%EC%96%B4%EB%96%BB%EA%B2%8C-%EB%9D%84%EC%9A%B8%EA%B9%8C-841e97dda1eb)*

#### [앱 2] 배민커넥트 (Baemin Connect) — 라이더 전용 B2B 필드워커 앱

**네비게이션 패턴**

```
┌────────────────────────────┐
│  배민커넥트   [알림🔔] [?] │  ← 미니멀 상단바
├────────────────────────────┤
│                            │
│  ┌──────────────────────┐  │
│  │  현재 상태: 대기중   │  │  ← 상태 뱃지 (크고 명확)
│  └──────────────────────┘  │
│                            │
│  ┌──────────────────────┐  │
│  │  ▶ 배달 시작하기     │  │  ← 풀-와이드 주요 CTA
│  └──────────────────────┘  │
│                            │
│  오늘 수익: ₩48,200        │  ← 보조 정보
│                            │
├────────────────────────────┤
│ 홈 | 내역 | 지도 | 더보기 │  ← 하단 탭바
└────────────────────────────┘
```

| 분석 차원 | 배민커넥트 패턴 |
|-----------|---------------|
| 네비게이션 | 하단 탭바 4개 + 풀-와이드 CTA 버튼 |
| CTA 위치 | 화면 하단 1/3 영역의 풀-너비(Full-width) 버튼 |
| 모달 | Sheet 형태 주로 사용 (주문 상세 정보) |
| 다크모드 | OS 연동 자동 지원 |
| 글자 크기 | 상태 텍스트 20sp, 수익 24sp (야외 가시성 고려) |
| 카드 스타일 | 어두운 배경 + 흰 텍스트 대비, 라이더 친화 고대비 |

**우리 제품 차용 패턴**
1. **풀-와이드 CTA**: "출근 체크인", "안전점검 시작" 등 핵심 행동을 화면 하단 풀-너비 버튼으로 배치 → 장갑 착용·야외·한 손 조작에 최적
2. **상태 뱃지 우선 표시**: 홈 진입 시 "오늘 출근 완료 / 안전점검 미완료" 상태를 색상 뱃지로 즉시 전달

*근거: 배민커넥트 Google Play 스토어 스크린샷 분석, [배민커넥트 앱 사용법 — oopy.io](https://woowabaebaek.oopy.io/38b8eee4-8c60-4c1d-9c43-89f7ba0b72e2)*

#### [앱 3] 쿠팡플렉스 (Coupang Flex) — 배송 알바 필드워커 앱

**네비게이션 패턴**

```
┌────────────────────────────┐
│  쿠팡플렉스          [메뉴]│  ← 단순 상단바 + 햄버거
├────────────────────────────┤
│  오늘 블록: 오전 9시-오후1시│
│  ┌────────────────────────┐ │
│  │ 배송지 목록 (리스트)  │ │  ← 단순 리스트 뷰
│  │ 1. ○○아파트 101호     │ │
│  │ 2. △△상가 2층         │ │
│  └────────────────────────┘ │
│  ┌────────────────────────┐ │
│  │  ✔ 배송 완료 처리    │ │  ← 액션 버튼 (비교적 큼)
│  └────────────────────────┘ │
└────────────────────────────┘
```

| 분석 차원 | 쿠팡플렉스 패턴 |
|-----------|--------------|
| 네비게이션 | 상단 햄버거 메뉴 (구형 패턴, 비권장) |
| CTA 위치 | 리스트 아이템 하단에 개별 버튼 |
| 글자 크기 | 14-16sp (상대적으로 소형) |
| 카드 스타일 | 리스트 셀 스타일 (카드형 아님) |

**시사점**: 쿠팡플렉스는 상단 햄버거 메뉴를 채용해 네비게이션이 숨겨져 있다. 필드워커 앱에서 이 패턴은 빠른 전환이 필요한 상황에서 불리하다. **반면교사 사례**.

**우리 제품 차용 패턴**
1. 네거티브 레슨: 햄버거 메뉴 대신 하단 탭바 채택이 필수 — 야외에서 빠른 탭 전환 보장
2. 배송 완료 처리처럼 **"한 번 탭으로 완료 처리"** 패턴은 적극 활용 (안전점검 체크 아이템별 체크박스)

*근거: [쿠팡플렉스 Google Play 앱](https://play.google.com/store/apps/details?id=com.coupang.flex.mobile&hl=en_US)*

## 2. 글로벌 트렌드 및 한국 적합도 평가

### 2.1 iOS 17+ / Material You (Material Design 3) 핵심 패턴

#### iOS Human Interface Guidelines 2024-2025 업데이트

| 패턴 | 내용 | 한국 적합도 |
|------|------|------------|
| **Dynamic Type** | 사용자 폰트 크기 설정을 앱이 자동 반영. SF Pro Text ≤19pt, SF Pro Display ≥20pt | ★★★★★ 고령 사용자 필수 |
| **Large Title** | 화면 상단 대형 타이틀로 섹션 명확화 | ★★★★☆ 한글 지원 良 |
| **Bottom Sheet / Detent** | iOS 16+에서 표준화. 하프 시트·풀 시트 전환 제스처 지원 | ★★★★★ 이미 한국 앱 표준 |

### 2.2 2024-2026 글로벌 모바일 UX 트렌드 — 한국 적합도 평가

#### 트렌드 1: Bento UI (벤토 그리드)

```
┌─────────────────────────────┐
│ ┌──────┐ ┌────────────────┐ │
│ │ GPS  │ │  오늘 일정     │ │
│ │체크인│ │  06:00 출근    │ │

│ ┌──────┐ └────────────────┘ │

│ │3건   │ │추천  │ │휴가  │ │
│ └──────┘ │경로  │ │신청  │ │
│          └──────┘ └──────┘ │
└─────────────────────────────┘
```

- **개념**: Apple iOS 17 위젯, Microsoft Fluent UI에서 채택. 비균일 크기 타일 그리드로 정보 계층 표현
- **특징**: 관련 콘텐츠를 모듈식 블록으로 그룹핑, 중요도에 따라 타일 크기 차등
- **한국 적합도**: ★★★★☆
- **적용 이유**: 홈 대시보드에 "출퇴근 상태(크게) + 안전점검·민원·경로(작게)"를 한 화면에 배치 가능. 정보 밀도와 접근성 균형
- **주의**: 타일이 너무 작아지면 50대 사용자 터치 정확도 저하 → 최소 2×2 그리드 이하 권장

*근거: [Bento Grids Galaxy UX Studio](https://www.galaxyux.studio/blog/bento-grids-the-new-standard-for-modular-ui-design/)*

#### 트렌드 2: Glassmorphism (글래스모피즘)

- **개념**: 반투명 유리 효과 레이어. 배경 블러(blur) + 투명도 조합
- **한국 현황**: 한국 주요 금융·커머스 앱에서 부분 채택 (토스 Sheet 배경 등)
- **한국 적합도**: ★★☆☆☆
- **야외 작업자 적합도**: 매우 낮음. 직사광선 환경에서 블러 처리된 배경은 텍스트 대비를 약화시켜 가독성 저하
- **결론**: 야외 작업자 앱에서 **적극 지양**. 오히려 단색 고-대비 배경이 적합

---

#### 트렌드 3: Microinteractions (마이크로인터랙션)

- **개념**: 버튼 탭 애니메이션, 로딩 상태 표시, 완료 체크마크 등 작은 피드백 효과
- **한국 적합도**: ★★★★★
- **야외 작업자 적합성**: 매우 높음
  - 장갑 착용 중 버튼이 실제로 눌렸는지 확인 어려움 → 즉각적 시각+햅틱 피드백 필수
  - 안전점검 체크 완료 시 초록 체크마크 애니메이션 → 성취감 + 실수 방지
- **구현 우선순위**: 출퇴근 체크인 완료, 안전점검 항목 체크, 민원 접수 완료 3개 필수

---

#### 트렌드 4: Neumorphism (뉴모피즘) — 잔재 평가

- **개념**: 소프트 UI, 같은 색상의 내·외 그림자로 볼록/오목 효과
- **한국 트렌드**: 2021-2022 피크, 현재는 사양
- **야외 작업자 적합도**: 매우 낮음. 저-대비 특성상 직사광선에서 버튼 식별 불가
- **결론**: **완전 배제**. 우리 앱에 부적합

---

#### 트렌드 5: AI 맞춤형 UI / 맥락 인터페이스

- **개념**: 사용자 행동·위치·시간 기반으로 UI가 자동 재배치
- **한국 적합도**: ★★★☆☆ (중장기 적용)
- **적용 예시**:
  - 오전 5:30~7:00: "출근 체크인" 버튼이 홈 최상단에 자동 노출

### 페르소나 A: 김민국 (Primary Persona) — 50대 베테랑 미화원

| 속성 | 내용 |
|------|------|
| **나이/성별** | 54세, 남성 |
| **근속** | 12년차 환경미화원 (현 구역 담당 8년) |
| **학력** | 고졸 |
| **디바이스** | 삼성 갤럭시 A34 (안드로이드), 5년 사용 |
| **디지털 수준** | 카카오톡·유튜브 능숙, 앱 설치·설정 어려움 |
| **신체** | 노안 시작 (안경 착용), 손가락 굵음 |
| **작업 환경** | 새벽 5-6시 출근, 야외, 장갑 착용, 직사광선 |
| **스마트폰 사용 패턴** | 주로 한 손 조작 (수거 작업 중), 짧은 시간 집중 |

**Primary JTBD**
> "나는 출근 직후 이것저것 확인해야 할 때, 빠르게 체크인하고 오늘 할 일을 한눈에 보고 싶다. 그래야 작업 지연 없이 바로 현장으로 나갈 수 있기 때문이다."

**Top 3 Pain Points**

1. **작은 글씨·버튼**: 안경 쓰고도 글자가 잘 안 보임. 버튼을 잘못 눌러 다시 처음부터 해야 할 때 좌절감 큼. (심각도: 높음)
2. **복잡한 메뉴 구조**: 어떤 메뉴에 뭐가 있는지 매번 기억해야 함. "저번에 어디서 했더라" 반복 경험. (심각도: 중간)
3. **네트워크 불안정**: 야외 음영지역에서 앱이 느리게 로딩되거나 오류 발생 시 당황. 재시도 방법 모름. (심각도: 높음)

**Top 3 Desired Gains**

1. **원-탭 완결**: 출근 체크인이 "GPS 확인 → 버튼 1번 → 완료" 3단계 이내로 끝나길 원함
2. **오늘 할 일 한눈에**: 앱 열자마자 "안전점검 해야 함 / 민원 1건 있음" 같은 요약을 바로 확인
3. **실수 복구**: 잘못 눌렀을 때 "취소" 또는 "뒤로 가기"로 쉽게 되돌아갈 수 있길 원함

**Unexpected Insight**
> 디지털 도구 자체에 대한 거부감은 없다. 오히려 "유튜브 잘 씀"에서 보이듯, 영상·시각 UI에 친숙하다. 문제는 **텍스트 밀도와 복잡한 폼**이다. 이미지·아이콘 중심 UI로 전환하면 학습 곡선이 급감한다.

**Product Fit Assessment**
- 강점: GPS 체크인, 안전점검 체크리스트 기능은 이 페르소나의 핵심 니즈에 직결
- 마찰: 폼 입력(민원 응답, 휴가 사유 입력)에서 이탈 위험 높음 → 텍스트 입력 최소화 필요
- 기회: 홈 화면 "오늘의 요약 카드" 구현 시 즉시 가치 체감 가능

### 페르소나 B: 박철수 — 40대 반장 (Primary + 관리 역할)

| 속성 | 내용 |
|------|------|
| **나이/성별** | 43세, 남성 |
| **직책** | 청소반 반장 (6명 팀 관리 + 본인 작업) |
| **근속** | 15년차 |
| **디바이스** | 삼성 갤럭시 S23 (안드로이드), 2년 사용 |
| **디지털 수준** | 스마트폰 능숙, 엑셀·카카오워크 사용 경험 |

| **스마트폰 사용 패턴** | 빈번한 앱 전환, 알림 즉시 확인 |

**Primary JTBD**
> "나는 작업 중 팀원 상황을 챙겨야 할 때, 내 작업 기록과 팀 현황을 동시에 빠르게 확인하고 싶다. 그래야 반장으로서 책임을 다하면서 내 업무도 놓치지 않을 수 있기 때문이다."

2. **팀원 현황 불투명**: 팀원이 안전점검 완료했는지 실시간으로 확인할 수 없음 — 전화로 직접 확인해야 함. (심각도: 높음)
3. **실적 입력 오류**: 구역·날짜·수거량 등 입력 필드 많아 실수 발생, 수정 후 재제출 번거로움. (심각도: 중간)

**Top 3 Desired Gains**

1. **대시보드형 팀 현황**: 팀원 6명의 오늘 체크인 상태·안전점검 완료 여부를 한 화면에서 파악
2. **빠른 컨텍스트 전환**: 민원 → 실적 → 출퇴근 화면을 2탭 이내로 오갈 수 있는 구조
3. **Smart Default**: 자주 입력하는 구역·수거량이 자동 기억되어 수정만 하면 완료

**Unexpected Insight**
> 반장 박철수는 본인이 관리자 기능을 더 원하지만, 실제로 가장 많이 쓰는 기능은 **자신의 출퇴근 체크인**이다. 팀 관리 기능이 아무리 좋아도 자신의 기본 업무 흐름이 방해받으면 즉시 이탈한다.

**Product Fit Assessment**
- 강점: 실적 관리·민원 화면은 이 페르소나 작업 다양성에 부합
- 마찰: 현재 앱에 팀 현황 대시보드가 없다면 반장 역할 지원 불충분
- 기회: 반장 전용 대시보드 탭 또는 위젯 추가 시 권한별 UX 분기 가능

### 페르소나 C: 이수진 — 30대 신입 여성 미화원

2. **오류 메시지 불친절**: "오류가 발생했습니다" 같은 메시지만 표시. 무엇이 잘못됐고 어떻게 해야 하는지 모름. (심각도: 높음)

2. **명확한 에러 처리**: "GPS 신호가 약합니다. 실외에서 다시 시도해 주세요" 같은 구체적 안내

**Product Fit Assessment**
- 강점: 기본 기능 구조는 단순해 빠른 학습 가능
- 마찰: 에러 상황·빈 상태(Empty State) 화면 UX 미비 시 신뢰도 하락
- 기회: 온보딩 투자 시 전체 사용자 이탈률 감소 (50대도 혜택)

## 5. UX 패턴 권고

### 5.1 "데스크톱 확대 느낌" 제거를 위한 5가지 핵심 원칙

현재 많은 모바일 ERP 앱이 데스크톱 화면을 그대로 축소한 형태여서 야외 작업자에게 극히 불편하다. 아래 5원칙은 이를 해결하는 핵심 처방이다.

---

#### 원칙 1: 수직 정보 계층화 (한 화면 = 한 작업)

**문제**: 데스크톱 ERP는 한 화면에 20개 이상 필드를 표시. 이를 모바일에 그대로 넣으면 스크롤이 과도하고 현재 위치를 잃음.

**해결**:
```
[잘못된 예]
┌─────────────────────────┐
│ 출근시간: [  입력  ]    │
│ 퇴근시간: [  입력  ]    │
│ 작업구역: [  선택  ]    │
│ 차량번호: [  입력  ]    │
│ 오늘날씨: [  선택  ]    │
│ 안전장비: □착용 □미착용 │
│ 특이사항: [  입력  ]    │
│ [제출]                  │
└─────────────────────────┘

[올바른 예: 단계별 분리]
Step 1/3 ━━━━━━━━
GPS로 위치 확인 중...
[출근 체크인] ← 풀-너비 CTA

(완료 후 자동으로 2단계로 이동)
Step 2/3 ━━━━━━━━━
오늘 안전 장비를 확인하세요
□ 안전조끼
□ 작업장갑  
□ 안전화
[다음 →]
```

**수치 목표**: 핵심 작업(출퇴근 체크인) 완료 탭 수 ≤ 3탭

---

#### 원칙 2: 엄지 영역 (Thumb Zone) 설계

**근거**: 연구에 따르면 사용자의 49%가 스마트폰을 한 손 엄지로만 조작. 화면 하단 2/3이 편안한 조작 영역(Green Zone).

```
┌─────────────────────┐
│  ░░░░░░░░░░░░░░░░   │ ← 상단 Red Zone: 탭 어려움
│  ░░░░░░░░░░░░░░░░   │   (헤더·날짜·상태 정보 배치)
├─────────────────────┤
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │
│  ▓ Yellow Zone    ▓ │ ← 보조 정보·리스트 배치
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │
├─────────────────────┤
│  ████████████████   │
│  █  Green Zone  █   │ ← 핵심 CTA·탭바 배치
│  ████████████████   │
│ [홈][출퇴근][안전][더보기] │ ← 탭바
└─────────────────────┘
```

**적용 규칙**:
- 탭바 항상 하단 고정 (iOS Safe Area + Android Nav Bar 고려)
- 1순위 CTA는 화면 하단 16-24dp 여백에 풀-너비 버튼
- 뒤로가기·닫기는 좌상단 (Red Zone) — 즉 실수 방지 목적으로 도달 어렵게

*근거: [Thumb Zone Mobile UX Elaris Software](https://elaris.software/blog/mobile-ux-thumb-zones-2025/), [하단 네비게이션 Medium](https://medium.com/@sjlim2/하단-네비게이션에-대해서-끄적끄적-703a6c270a8e)*

---

#### 원칙 3: 정보 밀도 최소화 — "카드 1개 = 메시지 1개"

**문제**: 테이블·그리드·대시보드를 그대로 모바일화하면 정보가 너무 작아지고 터치 타겟이 충돌.

**해결**: 카드 컴포넌트 설계 원칙

```
┌────────────────────────────┐
│ 🔴 안전점검 미완료         │ ← 아이콘 + 상태 텍스트 (18sp)
│ 오늘 작업 전 확인 필요     │ ← 설명 (14sp)
│                    [→ 이동]│ ← 우측 CTA
└────────────────────────────┘
최소 높이: 72dp
터치 타겟: 전체 카드 탭 가능
```

규칙:
- 카드 최소 높이 72dp (48dp 터치 타겟 + 여백)
- 카드 1개당 표시 정보 ≤ 3줄
- 리스트 아이템 간격 ≥ 8dp (터치 타겟 겹침 방지)

---

#### 원칙 4: 타이포그래피 — 야외 가독성 기준

**한글 가독성 연구 결과 (KCI 논문 기반)**:
- 모바일 최적 글자 너비: 넓게, 획 대비 줄이기, 자간 늘리기
- 산세리프(고딕체)가 세리프(명조)보다 모바일 가독성 우수
- 최소 본문 크기: 16px(웹) ≈ 16sp(Android)

```
[우리 앱 타이포그래피 스케일 제안]
─────────────────────────────────────
Pretendard (한글) / Roboto (숫자·영문)

스케일     크기   용도
─────────────────────────────────────
Display    28sp   홈 대형 상태 (출근 완료)
Headline   22sp   페이지 제목, 중요 수치
Title      18sp   카드 제목, 리스트 아이템
Body       16sp   본문, 설명 텍스트
Label      14sp   뱃지, 보조 정보
Caption    12sp   타임스탬프 (최소 사용)
─────────────────────────────────────
행간(line-height): 본문 기준 155% (네이버 연구)
예: 16sp × 1.55 = 25dp 행간
```

**야외 가독성 추가 고려**:
- 직사광선: 최소 대비비 4.5:1 (WCAG 2.1 Level AA), 권장 7:1
- 텍스트 + 배경 조합 금지: 연한 회색 텍스트 on 흰 배경 (대비비 2:1 이하)
- 권장 조합: 흰 배경 + #1A1A1A 텍스트 (대비비 18:1), 또는 네이비 배경 + 흰 텍스트

*근거: [모바일 UI 타이포그래피 가독성 — brunch](https://brunch.co.kr/@chulhochoiucj0/34), [모바일 텍스트 한글 폰트 가독성 향상 연구 KCI](https://www.kci.go.kr/kciportal/landing/article.kci?arti_id=ART002987878)*

---

#### 원칙 5: 단일 진실 소스 (Single Source of Truth) — 상태 기반 UI

**문제**: 사용자가 "내가 지금 출근한 건지, 안전점검이 됐는지" 를 앱의 여러 화면에서 각각 확인해야 함 → 혼란과 중복 작업 발생.

**해결**: 홈 화면을 "오늘의 작업 상태 허브"로 설계

```
[상태 기반 홈 화면 설계]

아침 5:30 (출근 전)          출근 후 (체크인 완료)
─────────────────            ─────────────────────
🔵 출근 체크인               ✅ 출근 06:12
[지금 출근하기 →]            ⚠️ 안전점검 미완료
                             [지금 완료하기 →]

안전점검 후 (모두 완료)       오후 3시 (퇴근 전)
──────────────────           ─────────────────────
✅ 출근 06:12                ✅ 출근 06:12
✅ 안전점검 완료              ✅ 안전점검 완료
📋 민원 1건 대기              📍 추천경로 확인
[민원 보기 →]                [퇴근 체크아웃 →]
```

## 6. 8개 메뉴 페이지별 적용 제안

> 아래 제안은 위 리서치 인사이트에서 직접 도출된 1개 핵심 패턴이다.

| 메뉴 | 핵심 UX 적용 제안 |
|------|-----------------|
| **홈 (Home)** | Bento 그리드 3-4 타일: "출퇴근 상태(크게) + 안전점검·민원·추천경로(작게)"로 오늘의 업무 상태를 첫 진입 즉시 한눈에 파악 |
| **출퇴근 (Check-in/out)** | GPS 반경 확인 → 풀-너비 CTA "출근하기" → 완료 시 강한 햅틱(CONFIRM) + 큰 체크마크 애니메이션으로 1-탭 완결 확인 |
| **안전점검 (Safety Check)** | 세로형 체크리스트(1화면 5-7항목, 56dp 행 높이) + 각 항목 탭 시 약한 햅틱(KEYBOARD_TAP) + 전체 완료 시 초록 진행 바 + Sheet로 "오늘 안전점검 완료!" 완료 피드백 |
| **실적 (Performance)** | 숫자 입력 전용 키패드 자동 노출 + "3.5톤" 등 단위 자동 붙이기 + Smart Default로 어제 수거량을 기본값으로 제안 |
| **휴가 (Leave)** | 잔여 연차·승인 규정을 신청 화면 상단 인-라인 카드로 표시 → 신청 중 별도 안내 페이지로 나가지 않아도 됨 |
| **민원 (Complaint)** | 민원 리스트에 왼쪽 스와이프 → "완료 처리" 퀵 액션 + 빈 상태(민원 없음) 화면에 안내 메시지로 업무 완료감 부여 |
| **프로필 (Profile)** | 개인 설정(글자 크기·다크모드·알림)을 최상단에 배치 (50대 사용자 글자 크기 조정 접근성 최우선) |
| **추천경로 (Route)** | 지도 위 경로 오버레이 + 하단 Sheet로 다음 목적지·예상 도착 시간 표시 + 내비 앱 바로 열기(카카오맵·네이버지도) 딥링크 1-탭 실행 |

## 7. 요약 및 우선순위 로드맵

### 7.1 핵심 발견 요약

```
[리서치 핵심 5개 발견]

1. 한국 필드워커 앱의 표준 수렴:
   → 하단 탭바(4-5개) + Bottom Sheet + 풀-너비 CTA

2. 40-60대 사용자의 가장 큰 진입 장벽:
   → 작은 글자(≤14sp) + 작은 터치 타겟(≤40dp) 조합

3. 야외 작업자에게 글래스모피즘·뉴모피즘은 금물:
   → 직사광선 대비비 저하로 화면 식별 불가

4. 경쟁사(WasteHero·AMCS·Routeware)는 글로벌 대형:
   → 한국어 + 로컬 법령(안전점검 의무) + 현장 인력 특화가 차별화

5. "데스크톱 확대 느낌" 해결 핵심:
   → 화면당 한 작업, 엄지 영역 설계, 상태 기반 홈 대시보드
```

5. GPS 확인 대기                5. 완료! 진동 + 체크 애니메이션
6. 확인 → 제출
─────────────────               ─────────────────────
총 6단계                        총 3단계 (50% 감소)
