## 1. 배경 및 문제

지자체 생활폐기물 수집운반은 위탁업체가 운영하지만 정산·보고 책임은 지자체에 있다. 현재 구조의 한계:

- **정산 비용**: 매월 25일 인건비 정산이 엑셀 수기 → 위탁업체 1인 3일 소요
- **감사 리스크**: 근태 수정이 엑셀에서 이루어져 변경 이력 추적 불가 → 노동청 감사 시 결손
- **권한 혼선**: 지자체가 위탁업체 데이터를 직접 수정할 수 있는 구조 → §7-1의 "지자체는 감독, 위탁업체는 운영" 원칙 위배
- **현장 입력 단절**: 근로자가 종이로 작성한 운행일지·민원·안전점검을 사무실에서 이중 입력 → 데이터 신뢰 저하

## 4. MVP 범위

### 4.1 한 줄 정의
**"멀티테넌트 기반 내부관리자/근로자 전용 핵심 4개 모듈(근태·민원·차량운행·원가정산) + 운영 대시보드"**

### 4.2 P0 기능 (MVP 필수, 10개)

1. 계정/권한 시스템 — Role 5종 정의, 미들웨어 RBAC 강제 (지자체는 GET-only)
2. 출퇴근 입력 + GPS 기록 (근로자 웹앱)
3. 근태 조정 + 감사 이력 영구 보존 (append-only)
4. 월 마감 확정 + 잠금
5. 결근 처리 (유/무급 구분, 인건비 공제)
6. 인건비 자동 산출 (연장 ×1.5 / 야간 ×0.5 추가 / 휴일 ×1.5~2.0)
7. 민원 접수 → 배정 → 처리 완료 워크플로우
8. 차량 운행일지 입력·승인
9. 원가 산정 자동 집계 (인건비 + 차량비 + 유류비 + 간접비)
10. 운영 대시보드 (오늘 근태 / 알림 / 원가 / 민원 / 차량)

### 4.3 비포함 (오버엔지니어링 제외)

| 항목 | 제외 사유 |
| --- | --- |
| 스마트워치 GPS 연동 | iOS/Android 네이티브 앱 선행, MVP 비용 과다 |
| 119 자동 긴급 문자 전송 | 통신사 API 계약·법적 책임 미정 |
| 건강기록카드 작성·관리 | 의료정보 별도 보호 체계 필요, 의료·노무 전문가 검토 선행 |
| 소급 수정 이중 승인 워크플로 엔진 | P1, 상태 관리 복잡성으로 MVP 후 도입 |
| 업체별 DB 스키마 분리 | 업체 10개 이상 확보 후 결정. MVP는 row-level 격리로 충분 |
| CC 인증 정식 취득 | 6~12개월 소요. MVP는 자체 OWASP Top 10 점검 보고서로 대체 |
| GPS 위변조 탐지 (Mock Location 차단) | 웹앱 단계 기술 한계, 1차는 GPS 이탈 플래그 모니터링 |

## 8. 성공 기준

### 8.1 시안(다음 단계)의 성공 기준 3가지

1. **5초 가치 인식** — 처음 여는 사용자가 설명 없이 "오늘 미출근 2명 / 처리기한 초과 민원 2건 / 정산 마감 5일"을 5초 안에 읽어낼 수 있다
2. **업무 흐름 일치** — KPI → 알림 → 근태 테이블 → 조치 버튼 순서가 관리자 오전 점검 동선과 일치한다
3. **개발 스펙 전달성** — 시안 HTML에서 디자인 토큰과 데이터 바인딩 포인트(필드명 수준)가 명확해 백엔드가 별도 설명 없이 API 명세를 뽑을 수 있다

### 8.2 Phase 1 풀 구현의 성공 기준 (참고)

- 인건비 정산 작업 시간 **3일 → 30분** 이하

- 지자체 행정감사용 보고서 **클릭 1회**로 PDF/Excel 다운로드
