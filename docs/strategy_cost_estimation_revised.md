# 원가산정 프로그램 전략 재설계
## — 기후환경에너지부 지침 기반 B2G·B2B 이중 전략

> 작성일: 2026-05-12  
> 배경: 생활폐기물 수집운반 원가산정은 "실제 원가 계산"이 아닌  
> **기후환경에너지부 지침 기반 예산설계 → 낙찰율 투찰 → 사후정산** 3단계 구조임을 반영

---

## 1. 도메인 구조 이해 — 전략의 출발점

### 1.1 실제 프로세스 흐름

```
[1단계: 예산설계]          [2단계: 입찰·계약]         [3단계: 사후정산]
                                                        
 지자체                      지자체 발주                 계약기간 종료 후
  ↓                           ↓                          ↓
 기후환경에너지부              업체들이 설계금액 추정      실제 집행액 확정
 지침에 따라                  → 낙찰율로 투찰가 결정     직접노무비
 설계금액 산출                (87~92% 구간 경쟁)         산업안전보건관리비
  ↓                           ↓                          감가상각비
 입찰공고                    낙찰업체 확정               → 지자체에 청구
                              낙찰금액으로 계약
```

### 1.2 세 단계의 핵심 특성

| 단계 | 주체 | 핵심 행위 | 데이터 성격 |
|------|------|---------|-----------|
| 예산설계 | 지자체 | 지침 적용 → 설계금액 산출 | 규정값·계수 기반 |
| 입찰·계약 | 업체 | 설계금액 역산 → 낙찰율 투찰 | 경쟁 정보·전략 |
| 사후정산 | 업체→지자체 | 실적 증빙 제출 → 차액 정산 | 운영 실적 데이터 |

### 1.3 사후정산 3대 항목

```
① 직접노무비
   - 계획: 지침 기준 인원 × 직종별 표준 임금단가
   - 실적: 실제 채용 인원 × 실제 지급 급여
   - 정산: 실적 > 계획 → 추가 지급 / 실적 < 계획 → 반환

② 산업안전보건관리비
   - 계획: 직접노무비의 일정 요율 (지침 규정)
   - 실적: 실제 안전교육·장비·TBM 등 집행액
   - 정산: 실제 집행액 기준

③ 감가상각비
   - 계획: 차량 대수 × 지침 기준 단가
   - 실적: 실제 차량 구입가 ÷ 내용연수(6년) 기준
   - 정산: 실제 감가상각액 기준
```

## 2. 고객군 재정의 — 두 개의 완전히 다른 니즈

### 고객 A: 지자체 (B2G)

**페인포인트**:
- 매년 원가산정 지침 개정 시 수작업으로 재계산
- 담당자 교체 시 노하우 단절
- 설계금액 오산정 시 예산 낭비 또는 업체 수익성 악화

**원하는 것**:
- 지침 변경 시 자동 반영된 설계금액 산출
- 인근 지자체 설계금액 비교 참고
- 낙찰율 이력 및 통계

---

### 고객 B: 위탁업체 (B2B)

**페인포인트**:
- 지자체 설계금액을 정확히 알 수 없어 투찰가 결정이 어려움
- 낙찰 후 사후정산 증빙 관리가 수작업(영수증·서류 더미)
- 직접노무비 초과 집행 시 손실 발생

**원하는 것**:
- 설계금액 역산 도구 (입찰공고 → 설계금액 추정)
- 과거 낙찰율 데이터 기반 투찰 전략
- 사후정산 3항목 증빙 자동화 (wci 운영 데이터 연동)

## 3. 원가산정 프로그램 기능 재설계

### 3.1 모듈 구조

```
cost.helpbiz.kr
├── [지자체 모드]  설계금액 산출 도구
│   ├── 지침 파라미터 관리 (매년 갱신)
│   ├── 설계금액 자동 계산
│   └── 인근 지자체 비교 리포트
│
├── [업체 모드]  입찰 전략 + 사후정산
│   ├── 설계금액 역산 추정기
│   ├── 낙찰율 분석 & 투찰가 시뮬레이터
│   └── 사후정산 3항목 관리
│
└── [공통]  입찰 이력 DB
    ├── 나라장터 낙찰 결과 수집
    └── 지역별·규모별 낙찰율 통계
```

### 3.2 모듈 A: 설계금액 산출 도구 (지자체용)

#### 입력 항목 (지침 변수)

```
구역 기본정보:
  - 수거구역 면적 (km²)
  - 대상 가구 수 (세대)
  - 연간 폐기물 발생량 (톤) — gis Ridge 모델 연동 가능

인력 기준:
  - 직종별 인원 수 (운전원·보조원·반장 등)
  - 지침 기준 직종별 임금단가 (매년 갱신)
  - 근무일수·근무시간

차량 기준:
  - 차종별 대수 (압축차·청소차·살수차 등)
  - 지침 기준 차종별 감가상각 단가
  - 연료 기준 소비량 + 단가

간접비 요율 (지침 고시값):
  - 산업안전보건관리비 요율
  - 일반관리비 요율
  - 이윤 상한 요율
```

#### 자동 산출 구조 (실제 엑셀 2026_pajusi_area10_cost.xlsx 검증 완료)

**직접노무비 — 직종별 1인당 월간 산출**:
```
■ 운전원·수거원 (대한건설협회 상반기 시중노임 — 보통인부)
  통상시급     = (기본급 ÷ 174h) + (환경미화수당 + 상여금) ÷ 209h
  기본급       = 일급 × 261일 ÷ 12개월
  주휴수당     = 일급 × 52일 ÷ 12개월
  연차수당     = 통상시급 × 8h × 15일 ÷ 12개월
  연장근무수당 = 통상시급 × 8h × 150% × 48일 ÷ 12개월
  환경미화수당 = 150,000원/월 (환경부 지침 고정)
  상여금       = 기본급 × 100% ÷ 12개월
  퇴직급여충당금 = (기본급 + 제수당 + 상여금) ÷ 12개월
  월간 직접노무비 = 기본급 + 제수당 + 상여금 + 퇴직급여충당금

■ 가로청소원 (중소기업중앙회 제조노임 — 단순노무종사원) — 기준 다름
  상여금 = 기본급 × 400% ÷ 12개월  (운전원 100%와 차이)

■ 2025년 기준 실적값
  직종        | 일급     | 월간 노무비  | 연간 노무비
  운전원·수거원 | 169,804 | 6,765,300  | 81,183,600
  작업반장     | 213,033 | 8,434,816  | 101,217,792
  가로청소원   |  90,830 | 4,191,305  |  50,295,660
```

**간접노무비 — 인적보험료 요율 (2025년 기준)**:
```
국민연금     = min(월노무비, 6,370,000) × 4.5%
건강보험     = 월노무비 × 3.545%
장기요양보험 = 건강보험료 × 12.95%
산재보험     = 월노무비 × 0.86%  (시설관리·사업지원서비스업)
고용보험     = 월노무비 × 1.15%
임금채권보장 = 월노무비 × 0.06%
석면피해구제 = 월노무비 × 0.006%
```

**경비 항목**:
```
감가상각비  ← 사후정산 대상
수리수선비
인적보험료  (위 요율 적용)
복리후생비  (물가정보·물가자료·시중가격 3소스 비교)
유류비
세금과공과
지급수수료
잡재료비
지급사용료
기타경비
마대구입비  (해당 구역만)
```

**최종 설계금액 산출**:
```
순 운영원가  = 노무비(직접+간접) + 경비 합계
일반관리비   = 순운영원가 × 7%
이  윤       = (순운영원가 + 일반관리비) × 약 10%
설계금액     = 순운영원가 + 일반관리비 + 이윤
```

#### 지침 자동 갱신 전략

```
매년 1분기 기후환경에너지부 고시 개정 시:
  → 관리자가 labor_wage_standards 테이블만 업데이트
  → 전국 모든 지자체 계산식 자동 반영

3년 계약 구조 대응:
  → 계약 시 3개 연차 동시 입력 (연도별 임금 단가만 다름)
  → 임금 고시 발표 시 해당 연차 자동 재산출
  
파라미터 테이블 (DB — 실제 검증값 반영):
  year | job_type    | wage_basis              | daily_wage
  2025 | 운전원·수거원 | 대한건설협회_보통인부_상반기 | 169,804
  2025 | 작업반장    | 대한건설협회_작업반장_상반기  | 213,033
  2025 | 가로청소원  | 중소기업중앙회_단순노무_연간  |  90,830
```

### 3.3 모듈 B: 낙찰율 분석 & 투찰가 시뮬레이터 (업체용)

#### 핵심 아이디어

지자체가 설계금액을 공개하지 않더라도,  
**입찰공고의 예정가격 범위 + 과거 낙찰 이력**으로 설계금액을 역산할 수 있다.

```
설계금액 역산 흐름:
  입찰공고 입력 (사업명·구역·인원·차량 수)
      ↓
  gis Ridge 모델 → 예상 폐기물 발생량
      ↓
  지침 파라미터 자동 적용 → 예상 설계금액 범위
      ↓
  과거 유사 사업 낙찰율 분포 조회
      ↓
  추천 투찰가 범위 (낙찰 확률 × 수익성 시뮬)
```

#### 낙찰율 시뮬레이터 UI

```
입력:
  예상 설계금액: 500,000,000원
  목표 낙찰율: [  87.5  ] %
  ───────────────────────────
  투찰가:       437,500,000원
  
  이 사업 유사 낙찰율 분포 (최근 3년):
  ████████████░░░ 85~88% 구간: 42%가 낙찰
  ░░██████████░░░ 88~91% 구간: 31%가 낙찰
  ░░░░░░░░████░░░ 91~93% 구간: 15%가 낙찰
  
  85% 투찰 시 → 수익성 시뮬:
    계약금액:  425,000,000원
    예상 원가: 411,000,000원  (사후정산 전)
    예상 이익:  14,000,000원  (3.3%)
```

#### 데이터 원천: 나라장터(G2B) 낙찰 결과

```
수집 항목:
  - 공고번호, 공고기관(지자체), 사업명
  - 예정가격, 낙찰금액, 낙찰율
  - 낙찰업체, 투찰 참가 업체 수
  - 계약일, 계약기간

gis 프로젝트의 기존 G2B API 수집 기능 활용 가능
```

### 3.4 모듈 C: 사후정산 3항목 관리 (업체용)

#### 사후정산의 흐름

```
계약체결
    ↓
계약기간 동안 wci가 실적 자동 수집:
  - 직접노무비: 근태집계 → 실제 지급 급여 증빙
  - 안전관리비: TBM 기록·안전장비 구매 영수증
  - 감가상각비: 차량 구입 증빙·감가상각 계산서
    ↓
계약기간 종료 → 정산서 자동 생성
    ↓
지자체 제출 → 차액 정산 (추가청구 or 반환)
```

#### 직접노무비 사후정산

```
계획 (계약 시):
  운전원 5명 × 임금단가 193,000원 × 250일 = 241,250,000원

실적 (wci 근태 데이터):
  실제 지급 급여 총액 (월별 집계)
  + 야간·연장·휴일 가산 실적
  = 실제 직접노무비

정산:
  실적 - 계획 = 차액 → 추가청구서 자동 생성
```

#### 산업안전보건관리비 사후정산

```
wci의 기존 데이터 연동:
  - TBM 실시 기록 (안전교육 횟수·참여인원)
  - SOS 발생 기록 (산재 관련 지출)
  - 안전장비 구매 영수증 (정비이력 첨부)

정산서 자동 생성:
  집행 항목 | 날짜 | 금액 | 증빙번호
  TBM 교육  | ...  | ...  | ...
  안전장갑  | ...  | ...  | ...
```

#### 감가상각비 사후정산

```
차량 마스터 데이터 입력:
  차량번호 | 구입일 | 구입가 | 차종 | 내용연수
  1234가   | 2023-03 | 45,000,000 | 압축차 | 6년

자동 계산:
  월별 감가상각액 = 구입가 ÷ (내용연수 × 12)
  계약기간 감가상각 총액 자동 집계

정산서: 차량별 감가상각 계산서 + 구입 영수증 자동 첨부
```

## 5. 전략과 전술

### 전략 1: 지자체를 관문으로 삼아 업체를 끌어들인다

**논리**:
- 지자체가 cost 도구로 설계금액을 산출
- 동일 도구를 쓰는 업체는 설계금액을 더 정확히 역산 가능
- "지자체가 쓰는 시스템"이라는 신뢰 효과

**전술**:
1. 지자체 담당자(환경과·자원순환과)에게 **무료 설계금액 산출 도구** 제공
2. 지자체가 입찰공고 등록 시 cost 링크 동봉 → 참가 업체에 노출
3. 업체는 낙찰율 분석 기능부터 유료화

---

### 전략 2: 지침 갱신을 경쟁력으로 만든다

**논리**:
- 현재 모든 지자체가 매년 지침 개정 시 수작업 재계산
- 담당자 교체 시 오류 발생 빈번
- 지침 파라미터 DB를 플랫폼이 관리하면 독점적 가치

**전술**:
1. 기후환경에너지부 고시 발표 즉시 파라미터 업데이트 (연 1회)
2. "올해 지침 반영 완료" 배너로 신뢰 구축
3. 5년 치침 이력 보관 → 과거 설계금액 재현 가능 (분쟁 시 근거)

---

### 전략 3: wci의 운영 데이터가 사후정산 증빙이 된다

**논리**:
- 현재 업체들은 사후정산 시 근태대장·운행일지·안전관리 서류를 수작업으로 취합
- wci를 쓰면 이 데이터가 이미 디지털로 존재
- 사후정산 증빙 자동화 = "wci를 써야 하는 강력한 이유"

**전술**:
1. wci 없이 cost 단독 판매 시: 수동 입력
2. wci + cost 패키지: 사후정산 자동화 활성화
3. 패키지 할인 → 번들 가입 유도

---

### 전략 4: gis의 낙찰율 DB가 업체에게 최고의 무기다

**논리**:
- 낙찰율은 모든 입찰 참가업체가 가장 원하는 정보
- 나라장터 공개 데이터지만 분석·시각화된 형태로는 없음
- gis의 G2B 수집 기능(기존 자산) 활용

**전술**:
1. gis에서 G2B 폐기물 입찰 공고·결과 자동 수집 (기존 기능 확장)
2. 지역별·규모별·연도별 낙찰율 분포 통계
3. 이 데이터를 cost 플랫폼의 프리미엄 기능으로 제공

## 6. 수익 모델 재설계

| 고객 | 기능 | 가격 | 과금 방식 |
|------|------|------|---------|
| **지자체** | 설계금액 산출 도구 | 무료 (도입기) / 연 30만원 | 연간 구독 |
| **지자체** | 지침 자동갱신 + 이력관리 | 연 100만원 | 연간 구독 |
| **업체** | 설계금액 역산 + 낙찰율 통계 | 월 10만원 | 월 구독 |
| **업체** | 투찰가 시뮬레이터 | 월 20만원 | 월 구독 |
| **업체** | 사후정산 자동화 (cost+wci 연동) | 월 30만원 | 월 구독 |
| **업체** | 풀패키지 (wci+cost+gis) | 월 50만원~ | 규모별 |

**수익 구조 특이점**:
- 지자체는 저가 또는 무료 → 채널 역할 (업체 유입 경로)
- 업체가 실제 수익원 → 입찰 1건당 수억원 계약이므로 월 10~50만원 SaaS 비용은 매우 낮은 허들

## 7. 개발 우선순위 재조정

### 1순위: 지침 파라미터 DB + 설계금액 산출 엔진

왜 1순위인가:
- 다른 모든 기능의 기반
- 지자체에 무료 제공 → 플랫폼 신뢰 구축
- 낙찰율 역산의 전제조건

구현 범위:
```
테이블: wage_guideline (년도·직종·일당)
테이블: vehicle_guideline (년도·차종·감가상각 단가)
테이블: rate_guideline (년도·항목·요율)
API:    POST /api/cost/calculate-design-amount
출력:   설계금액 계산서 PDF
```

### 2순위: G2B 낙찰 결과 수집 + 낙찰율 통계

왜 2순위인가:
- gis에 기존 G2B 수집 코드 존재 (재활용)
- 업체 유료화의 핵심 킬러 기능

### 3순위: 사후정산 3항목 자동화

왜 3순위인가:
- wci 데이터 연동 필요 (wci 먼저 안정화 후)
- 그러나 업체의 가장 큰 페인포인트 → 이게 완성되면 lock-in 효과

## 8. 요약 — 한 장 정리

```
원가산정의 진짜 의미:
  지침 → 설계금액(지자체) → 낙찰율(업체) → 사후정산(업체→지자체)

세 프로그램의 역할:
  gis      →  입찰 전 정보 제공 (예상 발생량, 낙찰율 DB)
  cost     →  설계금액 산출 + 투찰 전략 + 사후정산 관리
  wci      →  계약 이행 중 실적 수집 → 사후정산 증빙 공급

핵심 전략 4가지:
  ① 지자체 무료 진입 → 업체 유료화
  ② 지침 파라미터 DB = 플랫폼 해자(Moat)
  ③ wci 운영 데이터 = 사후정산 자동화 무기
  ④ G2B 낙찰율 DB = 업체 킬러 기능

개발 순서:
  1. 지침 파라미터 DB + 설계금액 산출 엔진
  2. G2B 낙찰율 수집 + 통계
  3. 사후정산 3항목 자동화 (wci 연동)
```
