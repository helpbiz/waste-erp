### 추가된 파일

|---|---|
| `lib/gps-adapters/etrace.ts` | ETRACE SEQ Pull 어댑터 + in-memory TTL 캐시 |
| `app/api/live-tracking/probe/route.ts` | SUPER_ADMIN 전용 API 응답 테스트 엔드포인트 |

### 변경된 파일
| 경로 | 변경 내용 |
|---|---|
| `prisma/schema.prisma` | `LiveTrackingConfig`에 `providerConfig Json?` 추가, `gisProvider` VarChar(20→30) 확장 |
| `app/api/live-tracking/config/route.ts` | gisProvider Zod enum에 `'etrace'` 추가 |
| `app/api/live-tracking/positions/route.ts` | `etrace` 브랜치 추가 — ETRACE API Pull, lastSeq DB 갱신, 차량번호 매칭 |
| `app/(admin)/super-admin/_super-admin-client.tsx` | `etrace` provider 옵션 추가, ETRACE 안내 패널, Base URL 자동 입력 |

### ETRACE API 정보
- Base URL: `http://ems25.etrace.co.kr/intf/`
- 인증: POST body `{ key: <apiKey> }`
- Endpoint 1: `POST /intf/pos_last_seq.jsp` → `{ msg: "ok", output: { SEQ: number } }`
- Endpoint 2: `POST /intf/pos_json.jsp` with `{ key, input: { SEQ } }` → `{ msg: "ok", positions: [...] }`
- 필드 매핑: `VEH_PLATES`→vehicleNo, `LAT`→lat, `LON`→lng, `DIRECTION`→heading, `SPEED`→speed
- SEQ 저장: `LiveTrackingConfig.providerConfig.lastSeq`

### 설정 방법 (대서환경)
1. 슈퍼어드민 → GIS 설정 탭
2. 대서환경(contractorId=4) 선택
3. Provider: `etrace` 선택 (Base URL 자동 입력됨)
4. API Key: ETRACE에서 발급받은 키 입력
5. 저장

### 동작 방식
- `GET /api/live-tracking/positions` 호출 시 ETRACE API Pull
- TTL 캐시(refreshSec초)로 과호출 방지
- 차량번호(VEH_PLATES) ↔ Vehicle.vehicleNo 매칭
- 미매칭 차량은 `noData: true`로 표시 (기본 CENTER 좌표 사용)

### VehicleGpsPosition 테이블 (local 모드용)
- vehicleId (unique) — 차량당 최신 위치 1건만 upsert
- lat, lng (Decimal 10,7)
- speed (km/h), heading (도)
- source (단말 식별자)
- updatedAt (자동 갱신)

> **참고**: `docker compose up -d app`은 `cleanerp-postgres` 컨테이너명 충돌로 실패함. `--no-deps` 옵션으로 postgres 재생성 없이 app만 재시작.

### 헬스체크 수정
Alpine 컨테이너 내부에서 `localhost` DNS 미해석 문제 → `docker-compose.prod.yml` 헬스체크 URL 수정:
```yaml
# 수정 전
test: ["CMD", "wget", "--spider", "-q", "http://localhost:3000/api/health"]
# 수정 후
test: ["CMD", "wget", "--spider", "-q", "http://127.0.0.1:3000/api/health"]
```
