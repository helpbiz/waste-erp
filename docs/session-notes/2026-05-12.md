## 1. 관리자 출퇴근 기능 (`/attendance`)

### 변경된 파일
| 경로 | 변경 내용 |
|---|---|
| `app/api/attendance/check-in/route.ts` | CONTRACTOR_ADMIN / INTERNAL_ADMIN 허용, lat/lng 선택사항, 관리자는 출퇴근 제한 규칙 · 지오펜스 미적용 |
| `app/api/attendance/check-out/route.ts` | 동일 — 퇴근도 관리자 역할 허용, GPS 선택사항, 시간제한 미적용 |
| `app/(admin)/attendance/page.tsx` | 오늘 날짜 관리자 본인 AttendanceRecord 조회 후 `selfRecord` prop으로 전달 |
| `app/(admin)/attendance/_attendance-client.tsx` | `AdminPunchWidget` 컴포넌트 추가 — 출근/퇴근 버튼, 브라우저 GPS 자동 수집(실패 시 좌표 없이 진행), 현재 상태 표시 |

### 동작 방식
- 관리자가 `/attendance` 접속 시 테이블 위에 "관리자 본인 출퇴근" 위젯 표시
- 출근 전: [출근] 버튼 → `POST /api/attendance/check-in` (좌표 있으면 전송, 없어도 허용)
- 출근 후: [퇴근] 버튼 → `POST /api/attendance/check-out`
- 퇴근 완료: "퇴근 완료" 뱃지 표시
- AttendanceRecord는 workerId = 관리자 userId 로 일반 근로자와 동일 테이블에 저장

## 2. TBM 사진 기능 추가

### 정책
- `TbmSession.content` (TEXT) 필드에 JSON 직렬화: `{ "text": "...", "photoDataUrl": "data:image/jpeg;base64,..." }`
- 기존 평문 content는 하위 호환 — JSON 파싱 실패 시 전체를 text로 사용
- DB 스키마 변경 없음

### 변경된 파일
| 경로 | 변경 내용 |
|---|---|
| `app/api/tbm/today/route.ts` | POST에 `photoDataUrl` 수락(최대 3MB base64), content+photo를 JSON으로 묶어 저장. GET 응답에 `content`(텍스트)·`photoDataUrl` 분리 반환 |
| `app/(admin)/safety/page.tsx` | TBM content JSON 파싱 후 `content`, `photoDataUrl`, `department` 별도 전달 |
| `app/(admin)/safety/_safety-client.tsx` | TbmInfo 타입에 `photoDataUrl`, `department` 추가. TbmWidget에 팀명 입력, 내용 입력, 사진 촬영(후면카메라, 800px JPEG 0.82 압축) 추가. 뷰 모드에서 팀명 뱃지 + 사진 표시 |
| `app/worker/safety/page.tsx` | TBM content JSON 파싱 후 `photoDataUrl` 분리 전달 |
| `app/worker/safety/_safety-worker-client.tsx` | TbmInfo 타입에 `photoDataUrl` 추가. TBM 뷰에 사진 표시 |

### 폼 필드 (관리자 TBM 편집)
1. 안전 주제 (필수, 2자 이상)
2. 팀명 (선택 — `TbmSession.department` 저장)
3. 간략 내용 (선택)
4. 사진 (선택 — `capture="environment"` 후면카메라, 800px JPEG 0.82 압축)

## 3. 차량 실시간 GPS 로컬 수신 모드

### 추가된 파일
| 경로 | 설명 |
|---|---|
| `app/api/live-tracking/gps-ingest/route.ts` | GPS 단말 Push 엔드포인트 |

### 변경된 파일
| 경로 | 변경 내용 |
|---|---|
| `prisma/schema.prisma` | `VehicleGpsPosition` 모델 추가, Vehicle·Contractor에 관계 추가 |
| `app/api/live-tracking/config/route.ts` | gisProvider Zod enum에 `'local'` 추가 |
| `app/api/live-tracking/positions/route.ts` | gisProvider='local'일 때 `VehicleGpsPosition` DB에서 실위치 조회 |
| `app/(admin)/live-vehicles/_live-vehicles-client.tsx` | local 모드일 때 인제스트 엔드포인트 정보 패널 표시 |

### GPS Ingest 엔드포인트
```
POST /api/live-tracking/gps-ingest?contractorId=<ID>
Authorization: Bearer <인제스트토큰>
Content-Type: application/json

{
  "vehicleNo": "12가3456",
  "lat": 37.4912,
  "lng": 127.0345,
  "speed": 30,
  "heading": 90
}
```

### 설정 방법 (대서환경)
1. GIS 설정 → `gisProvider=local`, `apiKey=<인제스트토큰>` 저장
2. GPS 단말에 위 엔드포인트 + 토큰 입력
3. 실시간 차량조회 페이지에서 위치 확인

### VehicleGpsPosition 테이블
- vehicleId (unique) — 차량당 최신 위치 1건만 upsert
- lat, lng (Decimal 10,7)
- speed (km/h), heading (도)
- source (단말 식별자)
- updatedAt (자동 갱신)
