## 1. contractor-org-master — 업체별 조직 마스터

### 배경
기존 PWA는 전역 `Position` 테이블만 있어 업체별 직책·직급을 관리할 수 없었다.

### 추가된 Prisma 모델
| 모델 | 설명 |
|---|---|
| `ContractorPosition` | 업체별 직책 (대표·팀장·기사 등) |
| `ContractorRank` | 업체별 직급 (1급~5급 등) |

`User`에 `contractorPositionId(FK)` + `rankId(FK)` 추가 (nullable, 기존 `positionId` 병존).

### 추가된 API
| 경로 | 메서드 | 설명 |
|---|---|---|
| `/api/contractor/positions` | GET, POST | 직책 목록 조회 + 신규 등록 |
| `/api/contractor/positions/[id]` | PATCH | 이름 변경·활성/비활성 (409: 사용 중 비활성화 차단) |
| `/api/contractor/positions/seed` | POST | 온보딩 시 기본 직책 8개 자동 생성 |
| `/api/contractor/ranks` | GET, POST | 직급 목록 조회 + 신규 등록 |
| `/api/contractor/ranks/[id]` | PATCH | 동일 패턴 |

### 추가된 UI
- `app/(admin)/users/_org-settings-tab.tsx` — 조직 설정 탭 (직책·직급 CRUD)
- `_users-client.tsx` 탭 확장 — CONTRACTOR_ADMIN·SUPER_ADMIN에게 "조직 설정" 탭 노출
- `ProfileEditor` — 사용자 프로필 편집 시 업체 직책·직급 선택 드롭다운 추가

### 기본값 상수 (`lib/org-defaults.ts`)
```
DEFAULT_POSITIONS: 대표, 이사, 팀장, 반장, 기사, 환경미화원, 사무원, 현장소장
DEFAULT_RANKS: 1급~5급
```

### 데이터 이전 유틸 (`lib/migrate-org-data.ts`)
기존 `User.positionId`(전역) → `contractorPositionId`, `User.rank`(String) → `rankId` 자동 매핑.

## 3. avac-facility-ops — AVAC 전용 모듈

### 3-1. AVAC Feature Flag

`ContractorFeature` 테이블의 `featureKey='avac'` 로 업체별 ON/OFF.

**추가 내용 (`lib/features.ts`)**
- `FeatureKey`에 `'avac'` 추가
- `FEATURE_CATALOG`에 AVAC 항목 등록 (기본 OFF)
- `isAvacContractor(contractorId)` 편의 함수
- `getAvacFacilities(contractorId)` — AVAC 업체 소속 시설 목록

**`lib/feature-packages.ts`**  
4개 요금제 패키지(TRIAL·BASIC·STANDARD·PRO) 모두에 `avac: false` 추가.

**AVAC 활성화 방법**  
슈퍼관리자 콘솔 → `🎛 회사별 기능 권한` → 해당 업체 → `avac` 토글 ON

### 3-2. TBM 시설별 전환

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| TBM 단위 | 업체(contractorId) + 날짜 | 업체 + **시설(facilityId)** + 부서 + 날짜 |
| 고유 제약 | `@@unique([contractorId, sessionDate])` | 제거 → `@@index` (시설별 복수 TBM 허용) |
| 조회 방식 | `findUnique(contractorId_sessionDate)` | `findFirst({ contractorId, facilityId, sessionDate })` |

**스키마 변경 (`prisma/schema.prisma`)**
```prisma
model TbmSession {
  facilityId   BigInt?  // NULL = 전사 TBM (비-AVAC 하위 호환)
  department   String?  // "운영팀", "관리팀" 등
  ...
}
```

**수정된 파일**
- `app/api/tbm/today/route.ts` — `?facilityId=N` 쿼리 지원, POST에 facilityId+department 수용
- `app/api/tbm/sign/route.ts` — 서명 시 facilityId 전달
- `app/(admin)/safety/page.tsx` — TBM 조회 findUnique → findFirst
- `app/worker/safety/page.tsx` — 동일

> 비-AVAC 업체는 `facilityId=null`로 기존과 동일하게 동작 (하위 호환).
