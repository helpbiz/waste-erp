### 1. 관리자 출퇴근 제한설정 화면 오류

**증상** 출퇴근 제한설정 페이지 진입 시 화면 렌더링 실패

**원인** 운영 DB `punch_restrictions` 테이블에 `sort_order` 컬럼 누락.  
Prisma 스키마에는 `sortOrder Int @default(0) @map("sort_order")` 로 정의되어 있으나 운영 환경에 마이그레이션 미적용.  
`orderBy: [{ sortOrder: 'asc' }]` 쿼리 실행 시 런타임 오류 발생.

**조치** 운영 DB에 컬럼 직접 추가 (이미 완료, 재시작 불필요)
```sql
ALTER TABLE punch_restrictions
  ADD COLUMN IF NOT EXISTS sort_order INT NOT NULL DEFAULT 0;
```

### Let's Encrypt 인증서 발급 과정

**HTTP-01 챌린지 시도 → 실패**

포트 80 외부 트래픽이 이 서버의 nginx에 도달하지 않음 (라우터 포워딩 미설정).  
nginx 액세스 로그에 Let's Encrypt 검증 요청 미수신 확인.  
5회 실패로 1시간 속도 제한 적용.

**DNS-01 챌린지로 전환**

```
DNS: helpbiz.kr → ns1~4.hostingkr.net
```

acme.sh 설치 및 TXT 레코드 발급:
```bash
curl https://get.acme.sh | sh -s email=admin@helpbiz.kr
~/.acme.sh/acme.sh --issue -d wci.helpbiz.kr \
  --dns --server letsencrypt \
  --yes-I-know-dns-manual-mode-enough-go-ahead-please
```

hostingkr.net DNS 콘솔에 TXT 레코드 수동 추가:
```
레코드: _acme-challenge.wci.helpbiz.kr
값:     4HxlcP88eBKfm6YbSwXfmsxKlq_dQJEwqMi6JpY_u2I
```

DNS 전파 확인 후 갱신 명령으로 인증서 발급:
```bash
~/.acme.sh/acme.sh --renew -d wci.helpbiz.kr \
  --server letsencrypt \
  --yes-I-know-dns-manual-mode-enough-go-ahead-please
```

**인증서 배포**

`/opt/smartas/nginx/certs/wci/` 디렉토리가 root 소유라 직접 복사 불가.  
Alpine 컨테이너를 특권 모드로 실행해 복사:
```bash
docker run --rm \
  -v /opt/smartas/nginx/certs/wci:/dest \
  -v /home/user/.acme.sh/wci.helpbiz.kr_ecc:/src:ro \
  alpine sh -c "cp /src/fullchain.cer /dest/fullchain.pem && \
                cp /src/wci.helpbiz.kr.key /dest/privkey.pem && \
                chmod 644 /dest/fullchain.pem && chmod 600 /dest/privkey.pem"

docker exec smartas-nginx nginx -s reload
```

**자동 갱신 등록**
```bash
~/.acme.sh/acme.sh --install-cert -d wci.helpbiz.kr \
  --reloadcmd "docker run --rm \
    -v /opt/smartas/nginx/certs/wci:/dest \
    -v /home/user/.acme.sh/wci.helpbiz.kr_ecc:/src:ro \
    alpine sh -c 'cp /src/fullchain.cer /dest/fullchain.pem && \
                  cp /src/wci.helpbiz.kr.key /dest/privkey.pem && \
                  chmod 644 /dest/fullchain.pem && chmod 600 /dest/privkey.pem' \
    && docker exec smartas-nginx nginx -s reload"
```

### manifest crossorigin 문제 수정 (PWA 설치 최종 원인)

인증서를 교체한 후에도 홈 화면 추가 후 standalone 모드로 동작하지 않는 문제가 남아있었음.

**원인**: `metadata.manifest: '/manifest.json'` 사용 시 Next.js 14가 자동으로
`crossorigin="use-credentials"` 를 manifest 링크에 추가.
iOS Safari는 이 속성이 있으면 manifest 파싱을 거부 → standalone 설치 차단.

```html
<!-- 수정 전 (문제) -->
<link rel="manifest" href="/manifest.json" crossorigin="use-credentials"/>

<!-- 수정 후 (정상) -->
<link rel="manifest" href="/manifest.json"/>
```

**조치** `app/layout.tsx`:
- `metadata` 객체에서 `manifest` 필드 제거
- `<head>` 에 `<link rel="manifest" href="/manifest.json" />` 직접 선언
- `<link rel="apple-touch-icon" href="/icons/icon-180.png" />` 명시 추가 (180×180)

**검증 명령**:
```bash
curl -s http://localhost:3001/login | grep -o 'rel="manifest"[^>]*>'
# 정상: rel="manifest" href="/manifest.json"/>
```

## HTTP-01 완전 자동 갱신 전환

### 배경

DNS-01 수동 모드는 갱신마다 TXT 레코드를 수동으로 추가해야 하는 문제가 있어 HTTP-01 자동갱신으로 전환.

### 구성

외부 포트 80이 기존에 `192.168.1.2`(nginx)로 포워딩되어 있었고 다른 서비스가 운영 중.  
라우터 포워딩 변경 없이, `192.168.1.2`의 nginx에 ACME 챌린지 경로만 프록시하는 방식으로 해결.

**`192.168.1.2` nginx에 추가한 설정** (`wci-acme-proxy.conf`):
```nginx
server {
    listen 80;
    server_name wci.helpbiz.kr;

    location /.well-known/acme-challenge/ {
        proxy_pass http://192.168.1.20:80;
        proxy_set_header Host $host;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}
```

**acme.sh webroot 모드로 전환**:
```bash
# Le_Webroot를 dns → webroot 경로로 변경
sed -i "s|Le_Webroot='dns'|Le_Webroot='/opt/smartas/sites/helpbiz'|" \
  ~/.acme.sh/wci.helpbiz.kr_ecc/wci.helpbiz.kr.conf
```
