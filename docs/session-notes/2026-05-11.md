## 1. 결재관리 통합 허브 (`/approvals`)

### 배경
기존에는 휴가·근태·운행일지·안전보고서 결재가 각 메뉴에 분산되어 있어 관리자가 미결재 건을 한눈에 파악하기 어려웠다.

### 추가된 파일
| 경로 | 설명 |
|---|---|
| `app/(admin)/approvals/page.tsx` | 서버 컴포넌트 — 인증·권한 확인 |
| `app/(admin)/approvals/_approvals-client.tsx` | 4탭 결재관리 클라이언트 |
| `app/api/approvals/pending/route.ts` | GET — 미결재 건수 집계 (4종) |
| `app/api/approvals/list/route.ts` | GET `?tab=pending\|approved\|rejected\|history&search=` |
| `components/ApprovalBanner.tsx` | 로그인 시 팝업 + TTS 알림 |

### 기능
- 4탭: 결재대기 / 승인 / 반려 / 전체이력
- 개인명 검색 (250ms 디바운스)
- 종류 뱃지: 휴가·근태조정·운행일지·안전보고서
- 상세 모달에서 직접 승인/반려 처리 (서명 없이 관리자 즉시 처리)
- 안전보고서는 `reviewNote` 텍스트 입력 후 "검토 완료"
- 사이드바 뱃지: 미결재 총 건수 실시간 표시
- `ApprovalBanner`: 60초 폴링, sessionStorage 중복방지, `window.speechSynthesis` TTS

### 버그 수정 (결재 관련)
| 에러 | 원인 | 수정 |
|---|---|---|
| `stage1_position_required` | 직책 미등록 관리자가 1차 결재 시도 | `isAdminRole` 플래그로 stage1·2 권한 우회 |
| `signature_required` | 결재관리 페이지에서 서명 없이 승인 | 관리자 역할은 저장 서명 자동 시도, 없으면 서명 없이 처리 |
| 민원 처리완료 `invalid_request` | 워커 앱이 `note` 필드 전송, API는 `resolveNote` 기대 | Zod transform으로 양쪽 키 허용, 기본값 `'처리 완료'` |

### 변경된 파일
- `app/api/leave-requests/[id]/route.ts` — `isAdminRole` 추가, 서명 nullable 처리
- `app/api/complaints/[id]/complete/route.ts` — `note`/`resolveNote` 이중 키 허용
- `app/(admin)/layout.tsx` — 미결재 4종 카운트, 결재관리 메뉴 항목 추가
- `app/(admin)/dashboard/page.tsx` — 결재 대기 현황 패널 추가

- `components/GlobalNotifications.tsx` — `<ApprovalBanner />` 추가

### 폼 구성
| 카드 | 내용 |
|---|---|
| 기본정보 | 작성일자, 작성자(자동), 전일/금일누적거리, 운행거리 자동계산, 동승자(직원명단 멀티선택), 운행기간/GPS정차지점 |
| 주유/요소수 | 주유량(ℓ)/금액(원), 요소수(ℓ)/금액(원) — 단위 ℓ 특수문자 |
| 공공봉투 사용 | 30ℓ·50ℓ·75ℓ 각 장수, 합계 자동계산 |
| 차량점검 | 10개 항목 토글버튼 (양호/이상/수리점검, 세차여부 예/아니오), 이상 항목 자동 요약 |
| 차량정비이력 | 정비업체(datalist 자동완성), 정비내용, 수리비용(원), 영수증(거래명세표) 촬영(후면카메라, 1200px JPEG 압축) |
| 특이사항 | 자유 텍스트 |

### 구현 세부
- 차량 선택: 바텀시트 피커, "차량 변경" 버튼
- 동승자: 같은 업체 활성 WORKER 목록에서 체크박스 멀티선택, 칩 표시
- 영수증 촬영: `capture="environment"` 후면카메라, canvas 리사이즈(1200px), JPEG 0.82 압축
- 확장 데이터 저장: 기존 `routeDetail TEXT` 필드에 JSON 직렬화 (DB 스키마 변경 없음)
- 제출 흐름: POST `/api/vehicle-logs` (DRAFT) → POST `/api/vehicle-logs/{id}/submit` (SUBMITTED)
- 하단 고정 제출 버튼: `bottom: calc(4rem + env(safe-area-inset-bottom))` — 탭바 위 정확히 배치
- 워커 홈 그리드: "차량일지" 메뉴 카드 추가 (orange-600)

## 5. 카카오 알림톡 프로바이더

### 변경된 파일
- `lib/sms.ts` — `KakaoAlimtalkProvider` 클래스 추가, `getSmsProvider()` 에 `'kakao'` 옵션 등록

### 작동 방식
- Solapi API (`/messages/v4/send-many/detail`) + `kakaoOptions` 페이로드 사용
- 템플릿 변수 `#{message}` 하나로 기상알림 본문 전체 매핑
- `disableSms: false` — 카카오 미설치·차단 수신자 SMS 자동 fallback
- 현재 상태: `.env.prod`에 `SMS_PROVIDER=simulation` 유지, 아래 설정 완료 후 `kakao`로 변경

### 활성화 환경변수 (`.env.prod`)
```
SMS_PROVIDER=kakao
SOLAPI_API_KEY=<Solapi 콘솔에서 발급>
SOLAPI_API_SECRET=<Solapi 콘솔에서 발급>
SMS_FROM=<발신번호>
KAKAO_CHANNEL_ID=<카카오채널 pfId, 예: KA01PF...>
KAKAO_TEMPLATE_CODE=<알림톡 템플릿 코드, 예: KA01TP...>
```

### 사전 작업 절차
1. [Solapi](https://solapi.com) 가입 → API Key / Secret 발급
2. 카카오 비즈니스 채널 개설 → Solapi 채널 연결
3. 알림톡 템플릿 등록: 변수 `#{message}` 포함 자유형 템플릿, 카카오 심사 완료 후 `KAKAO_TEMPLATE_CODE` 입력
4. `.env.prod` 갱신 → 컨테이너 재시작
