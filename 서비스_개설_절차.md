## 0. 서비스 개요 + 도입 효과

### 0.1 CleanERP가 해결하는 문제

| 기존 방식 | CleanERP 도입 후 |
|---|---|
| 출퇴근 종이 기록 | 모바일 GPS 자동 기록 |
| 휴가 결재 종이 회람 | 11종 휴가 + 2단계 자동 결재 |
| 차량 운행일지 수기 | 디지털 + 실시간 차량 위치 |
| TBM 안전교육 종이 서명 | 모바일 전자서명 + 미서명 추적 |
| 민원 → 위탁업체 → 작업자 전화 응대 | 시민 신고 자동 → 위탁업체 → 작업자 |
| 지자체 보고 매월 엑셀 정리 | 통계/보고서 자동 생성 + 워터마크 서명 |
| 대형폐기물 빼기 앱 별도 운영 | 자동 import (cron) |
| GIS 경로 별도 시스템 | TSP 최적화 + OSRM 도로 스냅 통합 |

### 0.2 적용 대상

✅ **적합**:
- 시·구청 위탁계약 환경미화 업체
- 청소차 5대 이상 운영
- 직원 20명 이상

⚠️ **추가 검토 필요**:
- 직원 5명 미만 영세 업체 (요금제 별도 협의)
- 산업폐기물 전용 업체 (모듈 일부만 활용)

## 2. 단계: 신청 검토 + 견적 발송 (Day 1~2)

### 2.1 운영팀 내부 검토 항목

- [ ] 사업자등록증 진위 확인 (사업자등록 상태조회 → 국세청)
- [ ] 위탁계약서 유효성 (계약기간 + 관할지자체 매칭)
- [ ] 신용도 확인 (필요 시)
- [ ] 도입 적합성 평가 (0.2장 기준)
- [ ] 요금제 산정 (부록 B)

### 2.2 견적 발송

**발송 항목**:
- 정식 견적서 (PDF, 전자서명)
- 표준 계약서 초안
- 도입 일정 제안 (14영업일 기준)
- 부록 B (SLA)

**발송 채널**: 이메일 + 전화 follow-up

### 2.3 응답 대기 SLA

| 시점 | 액션 |

## 3. 단계: 도입 사전 미팅 (Day 3~4)

### 3.1 미팅 형식

| 형태 | 소요 |
|---|---|
| 화상 미팅 (Zoom/구글미트) | 1시간 |
| 현장 방문 미팅 | 2~3시간 (왕복 포함) |
| 데모 환경 시연 | 30분 (미팅 후반) |

### 3.2 미팅 안건 (표준 1시간)

```
00:00 — 인사 + 참석자 소개 (5분)
00:05 — 위탁업체 운영 현황 청취 (15분)
        - 직원·차량·관할 지역
        - 현재 사용 중인 시스템·도구
        - 가장 큰 페인포인트
00:20 — CleanERP 데모 시연 (15분)
        - 슈퍼관리자 콘솔
        - 모바일 워커앱 (실시간 차량조회 등)
        - 시민 민원앱
00:35 — Q&A (15분)
00:50 — 일정 + 다음 단계 합의 (10분)
```

### 3.3 미팅에서 확정해야 할 항목

- [ ] **관할 지자체** 명단 (1개 또는 다수)
- [ ] **차량 대수** (운행일지 자동 생성용)
- [ ] **직원 수 + 직책 구성** (대표/임원/팀장/운전원/수거원/...)
- [ ] **부서 구성** (총무·운전·수거·청소 등)
- [ ] **차고지 주소** (전체 차량 출발지)
- [ ] **결재 라인** (휴가/사고 보고 등)
- [ ] **데이터 마이그레이션 범위** — 기존 시스템 데이터 이전 여부
- [ ] **운영 전환 희망일**

### 3.4 미팅 후 산출물

```
회의록 (PDF) → 양측 확인 후 사인
+ 확정 일정표
+ 필요 자료 추가 요청 목록 (있을 경우)
```

## 4. 단계: 계약 체결 (Day 5)

### 4.1 표준 계약 항목

| 항목 | 내용 |
|---|---|
| 서비스 범위 | CleanERP 전체 모듈 (대시보드·민원·안전·실적·차량·휴가 등) |

| 데이터 보관 | 계약 기간 + 5년 (산업안전보건법) |
| 데이터 소유권 | **위탁업체에 귀속** — 계약 종료 시 인계 |
| 가용성 SLA | 99.5% 월간 (부록 B) |
| 응답 SLA | 장애 등급별 (부록 B) |
| 요금 | 월정액 + 사용량 (부록 B) |
| 분쟁 해결 | 한국상사중재원 |

### 4.2 계약 체결 방식

| 방법 | 장점 | 비고 |
|---|---|---|
| **전자 계약** (모두싸인 등) | 빠름·법적 효력 동일 | 권장 |
| 종이 계약 (왕복 우편) | 전통적 | 5~7일 소요 |
| 직접 방문 서명 | 즉시 | 거리 가까운 경우 |

### 4.3 계약 체결 즉시 시작 작업

운영팀에서 계약 체결 확인 즉시:
1. 5단계 (시스템 셋업) 작업 티켓 생성
2. 위탁업체 담당자에게 6단계용 자료 요청 메일 발송

## 5. 단계: 시스템 인스턴스 셋업 (Day 6~7)

### 5.1 운영팀 작업 — `/super-admin` 콘솔에서

#### 5.1.1 지자체 등록 (필요 시)

```
/super-admin → "지자체 관리" 탭
→ 검색에서 해당 지자체 확인
   - 이미 시드되어 있으면 (전국 226개 시군구는 사전 등록됨) → 5.1.2로
   - 없으면 → "+ 지자체 신규 등록"
     - 이름, 행정코드(5자리), 광역단체, 상태=ACTIVE
```

#### 5.1.2 지자체 권한 정책 설정

```
/super-admin → "지자체 권한 매트릭스" 탭
→ 해당 지자체 → "권한 편집"
→ 허용 화면: dashboard·complaints·reports·safety·... (기본값 OK)
→ 허용 보고서: 8종 모두 체크 (위탁업체 운영 기준)
→ 다운로드: exportEnabled=true, bulkExportEnabled=false
→ 저장
```

#### 5.1.3 위탁업체 등록

```
/users → 위탁업체 추가
→ 필수: 회사명, 사업자번호, 관할 지자체, 대표자명, 연락처
→ 차고지 주소 입력 → 자동 지오코딩 (OSM Nominatim)
→ 계약 시작·종료일 입력
→ 상태: ACTIVE
```

#### 5.1.4 CONTRACTOR_ADMIN 계정 발급

```
/users → 신규 사용자
→ Role: CONTRACTOR_ADMIN
→ 이름·이메일·임시 비밀번호 (랜덤 12자)
→ 위탁업체: 5.1.3에서 등록한 업체
→ 임시 비밀번호 안내 메일 발송 — 첫 로그인 시 강제 변경 안내 포함
```

### 5.2 인프라 설정 (필요 시)

대부분 멀티테넌트 단일 인스턴스로 처리되지만, 다음 경우 추가:

| 케이스 | 추가 작업 |
|---|---|
| 광역단체 단위 큰 업체 | 별도 `Contractor` 1곳 + 산하 부서로 분리 |
| 자체 도메인 요청 | 서브도메인 발급 (예: `<company>.cleanerp.kr`) |
| KMS 별도 키 요청 | KMS_PROVIDER에 새 키 추가 (Phase B+) |

## 8. 단계: 시범 운영 (Day 13)

### 8.1 시범 운영 목적

- 실제 운영 환경에서 1일간 동작 확인
- 작업자들 모바일 사용 적응
- 결재 흐름 실제 작동 확인
- 운영팀 상시 대기로 즉시 대응

### 8.2 시범 운영 체크리스트

```
[오전 6시 — 첫 출근]
- [ ] 작업자 5명 이상 출근 등록 성공
- [ ] GPS 위치 정상 기록
- [ ] 06:00 이후 지각 분류 정상

[오전 7~8시 — TBM]
- [ ] TBM 세션 등록 완료
- [ ] 작업자 모두 서명 완료
- [ ] 미서명자 자동 추적

[오전 9시 — 본격 운영]
- [ ] 차량 운행일지 등록 시작
- [ ] 시민 민원 1건+ 접수 정상
- [ ] 위탁업체 관리자 → 작업자 배정 동작

[오전 11시 — 휴가 결재 테스트]
- [ ] 작업자 1명 휴가 신청
- [ ] 팀장 결재 → 대표 결재 흐름 동작

[오후 — 실적 등록]
- [ ] 폐기물 처리 실적 등록
- [ ] 자원순환센터 반입실적

[오후 6시 — 퇴근]
- [ ] 퇴근 등록
- [ ] 일일 통계 자동 집계 확인
```

### 8.3 발견 이슈 처리

```
시범 운영 중 발생한 문제는:
1. 운영팀 대기 채널(전용 카카오톡 단체방)에 즉시 보고
2. Critical → 30분 내 hotfix
3. Major → 익일 오전 패치
4. Minor → 정식 운영 후 정기 패치
```

### 8.4 시범 운영 종료 회의

```
오후 5시 — 위탁업체 + 운영팀 회의 (30분)
1. 발견 이슈 리뷰
2. 정식 운영 GO/NO-GO 결정
3. NO-GO인 경우 → 추가 시범 1일
4. GO인 경우 → 다음 날 정식 개시
```

## 부록 E — 자주 묻는 질문

### Q1. 도입 기간을 14일보다 줄일 수 있나요?

A. 가능합니다. 다음 조건이 모두 충족되면 7~10일도 가능:
- 직원·차량이 30개 미만 (소규모)
- 데이터 마이그레이션 불필요 (기존 시스템 없음)
- 양측 일정 100% 우선순위

긴급 도입 별도 패키지 (3일 셋업) 협의 가능.

### Q2. 기존 시스템 데이터를 그대로 가져올 수 있나요?

A. CSV로 export 가능한 데이터는 가져올 수 있습니다. PDF·이미지 등은 첨부 파일로만.
복잡한 시스템(자체 ERP)에서 이전하는 경우 별도 마이그레이션 비용(100~500만원) 협의.

### Q3. 자체 도메인 사용이 가능한가요?

A. Pro 플랜 이상에서 가능. 예: `cleanerp.gangnam.go.kr`
Cloudflare Tunnel + 와일드카드 인증서로 셋업, 1회 100만원.

### Q4. 시민 민원앱은 시민이 별도로 다운로드하나요?

A. **아니요. 별도 다운로드 불필요.**
URL `https://wci.helpbiz.kr/citizen` 접속 시 PWA로 동작. 홈 화면에 추가하면 앱처럼 사용.
QR 코드 스티커 인쇄용 PDF 무료 제공.

### Q5. 작업자 모바일 사용에 거부감이 있는 직원이 있다면?

A. 다음 단계로 적응 지원:
1. **단체 교육 30분** — 5가지 핵심 기능만
2. **빠른 시작 카드** (라미네이트) 차량마다 비치
3. **카카오톡 헬프데스크** — 사진+질문 보내면 즉시 답변
4. **첫 주는 종이+모바일 병행** 운영 (옵션)

### Q6. 데이터는 우리 회사 데이터인가요?

A. **위탁업체에 100% 귀속**합니다. CleanERP 운영팀은:
- 운영 목적 외 접근 금지 (계약상)
- 통계 분석 시 익명화 처리
- 계약 종료 시 90일 이내 완전 삭제 (법령 의무 보관 제외)

### Q7. 정전·인터넷 끊김 시에도 사용 가능한가요?

A. 부분적으로 가능합니다.
- PWA 앱 셸은 오프라인에서도 열림 (Service Worker)
- 출퇴근·민원 등록은 인터넷 복구 후 자동 동기화 (Phase B 예정)
- 현재는 인터넷 필수 (LTE 권장)

### Q8. 다른 위탁업체에 우리 데이터가 보이지 않나요?

A. **완벽한 격리**가 보장됩니다.
- DB 쿼리 단위에서 `contractorId` / `municipalityId` 자동 필터
- 미들웨어 + RBAC 5단계 권한 검증
- 분기 1회 cross-tenant 누출 audit 실시
- audit_log로 모든 데이터 접근 기록 (5년 보존)

### Q9. 서비스가 갑자기 중단되면 어떻게 되나요?

A. SLA 99.5%~99.95% 보장 + 위반 시 크레딧 자동 적용.
12개월 누적 SLA 위반 시 위탁업체 **무위약금 해지권** 발생.
또한 **일일 백업** 정책으로 최대 24시간 데이터만 손실 가능 (RPO 24h).

### Q10. 우리 시청·구청과 연계해서 보고서를 자동으로 보낼 수 있나요?

A. 가능합니다. 다음 옵션:
1. **시청 담당자에게 MUNI_ADMIN 계정 발급** → 직접 조회 (기본)
2. **이메일 자동 발송** (월간 PDF 자동 첨부)
3. **시청 자체 시스템 API 연동** — 별도 협의 (Phase C)
